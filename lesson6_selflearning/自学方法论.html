第 6 课延伸 · 自学方法论

AI 时代，如何自学一个领域里的「一个模块」

不讲空泛原则，给出可直接照着执行的 5 套流程 + 一套综合十步法。适用：有基本 Claude 使用经验的成年人。

用 AI 扩大输入，用问题驱动学习，用项目形成闭环，用输出验证理解，用迁移沉淀能力。
5 套核心流程 10 步综合法 配套 Prompt 模板 完整案例 · AI Agent 任务拆解

核心矛盾：AI 让「获取答案」极其廉价，但理解、判断、沉淀反而更难——容易产生「AI 做出来了 = 我学会了」的错觉。下面所有流程都在对抗这个错觉。

五套流程总览

流程适合场景核心目标关键动作
一 · 地图式刚接触陌生模块先建立整体认知画地图、抓核心 20%
二 · 项目驱动想快速学会并能用用作品倒逼学习做最小项目、卡点即学
三 · 问题树为解决真实问题从问题倒推知识拆问题树、找关键节点
四 · 费曼输出想真正讲清楚用表达验证理解讲给三类人、模拟提问
五 · 专家陪练想从入门到高手用 AI 当教练训练诊断、追问、压力测试

流程详解

流程 1地图式自学

先画地图，再进入细节

适合：刚进入陌生模块，不知从何开始（特征工程、项目管理、AI Agent、供应链…）。

执行步骤
  1. 定义模块边界：是什么 / 属于哪个领域 / 解决什么 / 不解决什么 / 与相邻模块的区别
  2. 建立三层知识地图：概念层（术语原理）→ 方法层（流程工具）→ 应用层（案例场景）
  3. 找出核心 20%：掌握哪些能解决 80% 的问题；哪些初学者可暂时跳过
  4. 做概念对比：易混淆概念对，列「定义/相同点/不同点/适用场景/误区」
  5. 建立最小理解闭环：立刻拿一个真实小例子做一次操作（如把小项目拆成任务表）
  6. 沉淀一页纸：定义、位置、核心概念、关键方法、误区、案例、可解决的问题、下一步
关键 Prompt
PROMPT我想自学【某领域】中的【某模块】。请帮我建立知识地图： 1. 它在整个领域中的位置 2. 主要解决什么问题 3. 前置知识 4. 核心概念 5. 与相邻模块的区别 6. 常见应用场景 7. 学完后应具备什么能力
目标：从完全陌生 → 建立清晰认知框架。
流程 2项目驱动自学

不要学完再做，而是在做中学

适合：不想只学理论，想快速做出东西（AI 工具、数据分析、网页、投资模型）。

执行步骤
  1. 选一个最小项目：1–3 天完成初版。标准：足够小 / 有明确输入输出 / 能验证 / 贴近真实
  2. 倒推知识清单：问「完成这个项目需要哪些知识」，而不是「我要学什么」
  3. 拆成 5 个小里程碑：M1 需求 → M2 Prompt → M3 界面 → M4 接口 → M5 部署
  4. 卡点即时学习：遇到卡点再精准学习，避免陷入资料收集
  5. 每个卡点做知识卡片：卡点 / 原因 / 解决方法 / 关键代码或步骤 / 以后怎么办
  6. 做复盘：哪些是 AI 做的、哪些我真懂了、下次能否独立完成 60%
关键 Prompt
PROMPT我想通过做一个【项目名称】来自学【某模块】。请帮我倒推： 1. 需要掌握的知识点 2. 每个知识点的优先级 3. 哪些必须先学 4. 哪些可边做边查 5. 哪些可暂时跳过 6. 推荐的最小实现路径
核心判断：始终自问——「这是 AI 做出来的，还是我理解后做出来的？」 目标：从「知道」变成「能做」。
流程 3问题树自学

不从知识出发，而从问题出发

适合：为解决真实业务问题（用项目系统替代 Excel、判断一个项目是否值得投）。

执行步骤
  1. 定义真实问题：要有真实场景 / 明确痛点 / 可验证结果 / 决策价值
  2. 搭建问题树：把大问题拆成一级 / 二级 / 三级问题
  3. 把问题映射为知识点：每个问题对应要学的知识模块
  4. 优先解决最关键节点：找出最关键的 3 个节点，先解决它们
  5. 形成方案草稿：现状/收益/对象/工具/试点/分工/规则/复盘/风险/行动
  6. 用 AI 做反方质疑：让 AI 扮演反对者提 10 条意见，再逐条回应
关键 Prompt
PROMPT请把【我要解决的问题】拆成一棵问题树： 1. 从业务目标开始 2. 拆成一/二/三级问题 3. 标注每个问题需学习的知识模块 4. 标注最关键的问题 5. 给出学习顺序
目标：为解决真实业务问题而学习。
流程 4费曼输出自学

能讲出来，才是真的懂

适合：想讲清楚、教别人、写文章、做分享、给老板汇报。

执行步骤
  1. 选一个具体模块：越小越好（✗「我要讲 AI」→ ✓「用 Prompt 做深度阅读」）
  2. 用三种对象各讲一遍：小学生（是否真懂）/ 同行（专业准确）/ 老板（商业价值）
  3. 用四段结构表达：是什么 / 为什么重要 / 怎么用 / 边界与误区
  4. 加入例子和反例：生活化例子 + 商业例子 + 反例 + 易误解场景
  5. 模拟听众提问：让 AI 扮演小白/专家/老板/反对同事各提 5 个问题
  6. 形成 10 分钟分享稿：标题/开场/重要性/核心/案例/误区/应用/一句话总结
关键 Prompt
PROMPT请为【某知识点】生成： 1. 一个生活化例子 2. 一个商业例子 3. 一个反例 4. 一个容易误解的场景 5. 一个适合讲给老板的表达
目标：从「我懂了」变成「我能讲清楚」。
流程 5专家陪练自学

让 AI 不只是解释器，而是教练、考官、反方

适合：已入门、想持续提高（投资判断、AI 产品设计、交易策略、商业分析）。

执行步骤
  1. 设定 AI 角色：入门老师 / 专家顾问 / 反方质疑者 / 案例教练 / 考官
  2. 先做诊断测试：10 道由易到难的题，判断水平并指出漏洞
  3. 苏格拉底式追问：每次只问一个问题，回答后指出对错，再问更深的
  4. 案例推演：给一个真实复杂案例，你先判断，再让 AI 批改
  5. 反方压力测试：让 AI 当挑剔专家，找逻辑漏洞/过度假设/风险遗漏
  6. 建立能力评分表：概念/工具/案例/风险/表达/迁移（1–5 分）
  7. 阶段复盘：已掌握什么 / 最大 3 个漏洞 / 哪些是假懂 / 下阶段练什么
关键 Prompt
PROMPT请用苏格拉底式教学法训练我【某模块】。每次只问一个问题， 我回答后你指出哪里对、哪里不完整、哪里有误区， 然后问下一个更深入的问题。
目标：从「会用」走向「有判断力」。

如何选择 / 组合

你的目标推荐流程
完全陌生，先入门地图式
想快速做出东西项目驱动
解决工作中的真实问题问题树
想讲课 / 写文章 / 汇报费曼输出
想训练专家级判断力专家陪练

推荐配方

地图式（建结构）+ 项目驱动（成能力）+ 问题树（解真实问题）+ 费曼输出（验证理解）+ 专家陪练（练判断力）。

综合流程 · 自学模块「十步法」

如果只保留一套，用这套——把五套流程的精华串成一条完整闭环。

  1. 定位模块 · 属于哪个领域 / 解决什么问题 / 为什么值得学
  2. 建立地图 · 核心概念 / 前置知识 / 方法体系 / 应用场景
  3. 抓核心 20% · 最重要的 20% / 先学什么 / 暂不学什么
  4. 做概念对比 · 相似概念 / 边界 / 常见误区
  5. 选最小项目或真实问题 · 能否 1–3 天完成一个最小闭环
  6. 边做边学 · 遇卡点再查，不陷入资料收集
  7. 让 AI 反向追问 · 不直接给答案，连续提问检查是否真懂
  8. 做反例和边界训练 · 何时不适用 / 错误用法 / 失败案例
  9. 输出一页纸 · 定义 / 价值 / 方法 / 案例 / 误区 / 应用
  10. 迁移到真实场景 · 能否形成模板 / 工具 / 流程 / 方法论

完整案例 · 自学「AI Agent 中的任务拆解模块」

步骤执行内容
1 定位领域 AI Agent；模块 任务拆解；解决「把复杂目标拆成可执行步骤」
2 地图前置：Prompt/工作流/工具调用/上下文；核心：Goal/Task/Subtask/Plan/Tool/Feedback；方法：CoT / Plan-and-Execute / ReAct
3 核心 20%目标定义 → 任务拆解 → 步骤排序 → 工具匹配 → 执行反馈
4 对比Prompt（一次性）/ Workflow（固定流程）/ Agent（动态规划行动）
5 最小项目做一个「AI 学习计划生成器」：输入领域+模块 → 输出学习地图、7 天计划、练习、测试题
6 边做边学输出混乱→学结构化 Prompt；计划不合理→学拆解原则；无法检查→学评估指标
7 AI 追问为什么这样拆？前置条件？哪步最易失败？只有 2 小时怎么压缩？
8 反例目标太大 / 步骤不清 / 没顺序 / 没验证标准 / 没反馈机制
9 一页纸定义、价值、方法链、误区、应用一页沉淀
10 迁移迁移到项目管理拆任务、投研框架、教学路径、读书拆解

一句话总结

先建地图，再抓核心；先做闭环，再问边界；先讲出来，再迁移到真实场景。
