# Spec：AI 时代自学一个领域 / 一个模块的方法论

> 第 6 课延伸 · 自学方法论
> 主题：在 AI 时代，如何系统地自学一个领域里的「一个模块」
> 适用对象：成年人，有基本 Claude / Claude Code 使用经验
> 一句话：**用 AI 扩大输入，用问题驱动学习，用项目形成闭环，用输出验证理解，用迁移沉淀能力。**

---

## 0. 设计目标

这份 spec 不是讲原则，而是给出**可直接照着执行的流程**。

它要解决的核心矛盾是：AI 让「获取答案」变得极其廉价，但**理解、判断、沉淀**反而更难——因为很容易产生「AI 做出来了 = 我学会了」的错觉。

因此所有流程都围绕一条主线设计：

```
建立结构 → 形成能力 → 解决真实问题 → 验证理解 → 迁移沉淀
   地图       项目          问题树         费曼         陪练
```

---

## 1. 五套核心流程总览

| 流程 | 适合场景 | 核心目标 | 关键动作 |
|------|----------|----------|----------|
| 一、地图式自学 | 刚接触陌生模块 | 先建立整体认知 | 画地图、抓核心 20% |
| 二、项目驱动自学 | 想快速学会并能用 | 用作品倒逼学习 | 做最小项目、卡点即学 |
| 三、问题树自学 | 为解决真实问题 | 从问题倒推知识 | 拆问题树、找关键节点 |
| 四、费曼输出自学 | 想真正讲清楚 | 用表达验证理解 | 讲给三类人、模拟提问 |
| 五、专家陪练自学 | 想从入门到高手 | 用 AI 当教练反复训练 | 诊断、追问、压力测试 |

> 五者不是互斥的，**最佳实践是组合使用**（见第 8 节十步法）。

---

## 2. 流程一：地图式自学流程

**适合**：刚进入一个陌生模块，不知道从哪开始（如特征工程、项目管理、AI Agent、供应链、AIDC 方案）。
**核心思想**：先画地图，再进入细节。

### 步骤

1. **定义模块边界**：它是什么 / 属于哪个领域 / 解决什么 / 不解决什么 / 与相邻模块的区别。
2. **建立三层知识地图**：
   - 概念层：核心术语、定义、原理
   - 方法层：常见流程、工具、方法论
   - 应用层：案例、场景、真实问题
3. **找出核心 20%**：掌握哪些能解决 80% 的问题；哪些初学者可暂时跳过。
4. **做概念对比**：整理最易混淆的概念对，列「定义 / 相同点 / 不同点 / 适用场景 / 新手误区」。
5. **建立最小理解闭环**：立刻拿一个真实小例子做一次操作（如把一个小项目拆成任务表）。
6. **沉淀一页纸**：定义、位置、核心概念、关键方法、误区、案例、可解决的问题、下一步。

### 关键 Prompt

```
我想自学【某领域】中的【某模块】。请帮我建立知识地图：
1. 它在整个领域中的位置
2. 主要解决什么问题
3. 前置知识
4. 核心概念
5. 与相邻模块的区别
6. 常见应用场景
7. 学完后应具备什么能力
```

```
请整理【某模块】中最易混淆的 10 组概念，每组用表格说明：
各自定义 / 相同点 / 不同点 / 适用场景 / 新手常见误区。
```

**适合目标**：从完全陌生 → 建立清晰认知框架。

## 3. 流程二：项目驱动自学流程

**适合**：不想只学理论，想快速做出东西（AI 工具开发、数据分析、网页开发、投资模型）。
**核心思想**：不要学完再做，而是在做中学。

### 步骤

1. **选一个最小项目**：1–3 天能完成初版。标准：足够小 / 有明确输入 / 有明确输出 / 能验证 / 贴近真实。
2. **倒推知识清单**：问「完成这个项目需要哪些知识」，而不是「我要学什么」。
3. **拆成 5 个小里程碑**：每完成一个就有一次成就感（M1 需求 → M2 Prompt → M3 界面 → M4 接口 → M5 部署）。
4. **卡点即时学习**：遇到卡点再精准学习，避免陷入资料收集。
5. **每个卡点做知识卡片**：卡点 / 原因 / 解决方法 / 关键代码或步骤 / 以后怎么办。
6. **做复盘**：我完成了什么 / 学会了什么 / 哪些是 AI 做的 / 哪些我真懂了 / 下次能否独立完成 60%。

### 关键 Prompt

```
我想通过做一个【项目名称】来自学【某模块】。请帮我倒推：
1. 需要掌握的知识点  2. 每个知识点的优先级
3. 哪些必须先学  4. 哪些可边做边查  5. 哪些可暂时跳过
6. 推荐的最小实现路径
```

**核心判断**：始终自问——「这是 AI 做出来的，还是我理解后做出来的？」

**适合目标**：快速从「知道」变成「能做」。

## 4. 流程三：问题树自学流程

**适合**：为解决真实业务问题而学（如用项目系统替代 Excel、判断一个项目是否值得投）。
**核心思想**：不从知识出发，而从问题出发。

### 步骤

1. **定义真实问题**：要有真实场景 / 明确痛点 / 可验证结果 / 决策价值。
   - ✗「我要学项目管理」 → ✓「我想解决团队进度混乱、责任不清、Excel 更新不及时」
2. **搭建问题树**：把大问题拆成一级 / 二级 / 三级问题。
3. **把问题映射为知识点**：每个问题对应要学的知识模块。
4. **优先解决最关键节点**：找出最关键的 3 个节点，先解决它们。
5. **形成方案草稿**：输出一份可落地的方案（现状 / 收益 / 对象 / 工具 / 试点 / 分工 / 规则 / 复盘 / 风险 / 行动）。
6. **用 AI 做反方质疑**：让 AI 扮演反对者提 10 条反对意见，再逐条回应。

### 关键 Prompt

```
请把【我要解决的问题】拆成一棵问题树：
1. 从业务目标开始  2. 拆成一/二/三级问题
3. 标注每个问题需学习的知识模块  4. 标注最关键的问题  5. 给出学习顺序
```

```
请扮演一个反对【某方案】的团队成员，从成本、学习负担、执行阻力、
重复录入、老板不看、团队不配合等角度提 10 个反对意见，然后帮我逐条回应。
```

**适合目标**：为解决真实业务问题而学习。

## 5. 流程四：费曼输出自学流程

**适合**：想讲清楚、教别人、写文章、做分享、给老板汇报。
**核心思想**：能讲出来，才是真的懂。

### 步骤

1. **选一个具体模块**：越小越好（✗「我要讲 AI」 → ✓「我要讲如何用 Prompt 做深度阅读」）。
2. **用三种对象各讲一遍**：
   - 小学生版 → 检查是否真懂
   - 同行版 → 检查专业准确性
   - 老板 / 客户版 → 检查商业价值
3. **用四段结构表达**：是什么 / 为什么重要 / 怎么用 / 边界与误区。
4. **加入例子和反例**：生活化例子 + 商业例子 + 反例 + 易误解场景。
5. **模拟听众提问**：让 AI 扮演小白 / 专家 / 老板 / 反对同事各提 5 个问题，回答不清处即理解漏洞。
6. **形成 10 分钟分享稿**：标题 / 开场问题 / 重要性 / 核心概念 / 案例 / 误区 / 应用 / 一句话总结。

### 关键 Prompt

```
请为【某知识点】生成：
1. 一个生活化例子  2. 一个商业例子  3. 一个反例
4. 一个容易误解的场景  5. 一个适合讲给老板的表达
```

**适合目标**：从「我懂了」变成「我能讲清楚」。

## 6. 流程五：专家陪练自学流程

**适合**：已入门，想持续提高（投资判断、AI 产品设计、交易策略、商业分析）。
**核心思想**：让 AI 不只是解释器，而是教练、考官、反方、专家团。

### 步骤

1. **设定 AI 角色**：入门老师 / 专家顾问 / 反方质疑者 / 案例教练 / 考官。
2. **先做诊断测试**：让 AI 出 10 道由易到难的题，判断你是入门/初级/中级/高级，并指出漏洞。
3. **苏格拉底式追问**：AI 每次只问一个问题，回答后指出对错与不完整，再问更深的问题。
4. **案例推演**：给一个真实复杂案例，你先判断，再让 AI 批改。
5. **反方压力测试**：让 AI 当挑剔专家，找逻辑漏洞 / 过度假设 / 风险遗漏 / 数据不足 / 执行难点。
6. **建立能力评分表**：概念理解 / 工具使用 / 案例分析 / 风险判断 / 表达输出 / 迁移应用（1–5 分）。
7. **阶段复盘**：已掌握什么 / 最大的 3 个漏洞 / 哪些是假懂 / 下阶段练什么 / 设计 3 个进阶练习。

### 关键 Prompt

```
请用苏格拉底式教学法训练我【某模块】。每次只问一个问题，
我回答后你指出哪里对、哪里不完整、哪里有误区，然后问下一个更深入的问题。
```

```
请作为一个非常挑剔的专家审查我的方案，重点指出：
逻辑漏洞 / 过度假设 / 风险遗漏 / 数据不足 / 执行难点 / 可能失败的地方。
```

**适合目标**：从「会用」走向「有判断力」。

## 7. 如何选择流程

| 你的目标 | 推荐流程 |
|----------|----------|
| 完全陌生，先入门 | 地图式自学 |
| 想快速做出东西 | 项目驱动自学 |
| 解决工作中的真实问题 | 问题树自学 |
| 想讲课 / 写文章 / 汇报 | 费曼输出自学 |
| 想训练专家级判断力 | 专家陪练自学 |

**推荐的综合配方：**

```
地图式（建结构）+ 项目驱动（成能力）+ 问题树（解真实问题）
+ 费曼输出（验证理解）+ 专家陪练（练判断力）
```

## 8. 综合流程：AI 时代自学模块「十步法」

> 如果只保留一套，用这套。它把五套流程的精华串成一条完整闭环。

1. **定位模块**：属于哪个领域 / 解决什么问题 / 为什么值得学。
2. **建立地图**：核心概念 / 前置知识 / 方法体系 / 应用场景。
3. **抓核心 20%**：最重要的 20% / 先学什么 / 后学什么 / 暂不学什么。
4. **做概念对比**：相似概念 / 边界 / 常见误区。
5. **选最小项目或真实问题**：能否 1–3 天完成一个最小闭环。
6. **边做边学**：遇卡点再查，不陷入资料收集。
7. **让 AI 反向追问**：不直接给答案，连续提问检查是否真懂。
8. **做反例和边界训练**：何时不适用 / 错误用法 / 失败案例。
9. **输出一页纸**：定义 / 价值 / 方法 / 案例 / 误区 / 应用。
10. **迁移到真实场景**：能否用于工作、投资、教学、产品，能否形成模板 / 工具 / 流程 / 方法论。

## 9. 完整案例：自学「AI Agent 中的任务拆解模块」

| 步骤 | 执行内容 |
|------|----------|
| 1 定位 | 领域 AI Agent；模块 任务拆解；解决「把复杂目标拆成可执行步骤」 |
| 2 地图 | 前置：Prompt / 工作流 / 工具调用 / 上下文；核心：Goal/Task/Subtask/Plan/Tool/Feedback；方法：CoT / Plan-and-Execute / ReAct |
| 3 核心 20% | 目标定义 → 任务拆解 → 步骤排序 → 工具匹配 → 执行反馈 |
| 4 对比 | Prompt（一次性）/ Workflow（固定流程）/ Agent（动态规划行动） |
| 5 最小项目 | 做一个「AI 学习计划生成器」：输入领域+模块 → 输出学习地图、7 天计划、练习、测试题 |
| 6 边做边学 | 输出混乱→学结构化 Prompt；计划不合理→学拆解原则；无法检查→学评估指标 |
| 7 AI 追问 | 为什么这样拆？前置条件？哪步最易失败？只有 2 小时怎么压缩？ |
| 8 反例 | 目标太大 / 步骤不清 / 没顺序 / 没验证标准 / 没反馈机制 |
| 9 一页纸 | 定义、价值、方法链、误区、应用一页沉淀 |
| 10 迁移 | 迁移到项目管理拆任务、投研框架、教学路径、读书拆解 |

## 10. 一句话总结

> **先建地图，再抓核心；先做闭环，再问边界；先讲出来，再迁移到真实场景。**

更简洁地说：

> 用 AI 扩大输入，用问题驱动学习，用项目形成闭环，用输出验证理解，用迁移沉淀能力。
