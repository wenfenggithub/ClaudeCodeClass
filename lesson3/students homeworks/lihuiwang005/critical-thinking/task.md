# 批判性思维学习网页 Task

## 1. 项目阶段总览

本项目拆分为 4 个阶段推进：

1. 内容与信息架构阶段
2. 页面基础实现阶段
3. 交互与学习体验增强阶段
4. 扩展能力与验收阶段

阶段推进原则：
- 优先完成 `spec.md` 中的 P0 功能
- 在页面主流程稳定后补充 P1 交互与学习体验
- P2 功能作为后续扩展，不阻塞首版上线
- 每个阶段完成后都应有可检查的交付物

## 2. 阶段一：内容与信息架构

### 阶段目标

明确页面要展示的核心内容、资料结构和信息层级，为后续页面设计与开发提供稳定输入。

### 任务 1.1：整理核心概念内容

目标：
- 明确“批判性思维”的定义、边界和一句话总结
- 区分批判性思维与质疑、抬杠、怀疑主义的关系

交付物：
- 概念总览内容草稿
- 关键概念解释列表
- 正确理解与常见误解对照表

验收标准：
- 用户能在 1-2 分钟内理解批判性思维的基本含义
- 每个关键概念都有简洁解释
- 至少包含 4 条常见误解及其澄清

### 任务 1.2：整理核心方法框架

目标：
- 将批判性思维拆分为可学习、可展示、可练习的方法步骤

交付物：
- 提问、分解、比较、评估、反思五个方法模块
- 每个方法模块包含定义、使用场景和示例问题

验收标准：
- 每个方法模块都能独立展示
- 方法之间有清晰顺序或关系
- 用户能基于方法模块形成一套简单检查清单

### 任务 1.3：整理案例分析内容

目标：
- 准备覆盖不同场景的案例，用于展示批判性思维的实际应用

交付物：
- 至少 4 个案例草稿
- 每个案例包含场景背景、表面结论、可疑点、关键证据、推理过程、最终判断

验收标准：
- 案例覆盖新闻信息、社交媒体、消费决策、工作方案评估
- 每个案例都能支持逐层揭示式展示
- 案例内容避免依赖无法验证的事实

### 任务 1.4：整理外部资料与名言

目标：
- 准备可在页面中展示的外部资料和名人名言

交付物：
- 外部资料清单
- 名人名言清单
- 每条内容的来源、用途标签和待确认状态

验收标准：
- 外部资料至少覆盖入门、深入、案例、练习四类
- 每条资料包含标题、来源、摘要、链接或待确认说明
- 名人名言包含作者、出处或“待确认”标记

## 3. 阶段二：页面基础实现

### 阶段目标

完成首版网页的主体结构和 P0 内容展示，让用户可以完整浏览主要学习内容。

### 任务 2.1：搭建页面整体结构

目标：
- 建立单页滚动式页面结构和基础导航

交付物：
- 顶部导航或侧边导航
- 概念总览、核心方法、误区辨析、案例分析、外部资料等基础区块
- 页面锚点结构

验收标准：
- 用户可以从导航跳转到各主要模块
- 页面模块顺序与 `spec.md` 信息架构一致
- 页面在桌面端和移动端都能正常浏览

### 任务 2.2：实现概念总览与误区辨析区块

目标：
- 让用户快速理解项目主题和关键边界

交付物：
- 概念总览区块
- 一句话总结
- 正确理解与常见误解对照区块

验收标准：
- 内容层级清晰
- 对照信息易于扫描
- 不出现大段堆叠文本

### 任务 2.3：实现核心方法区块

目标：
- 展示批判性思维的核心方法路径

交付物：
- 提问、分解、比较、评估、反思五个方法展示组件
- 每个方法的说明与示例问题

验收标准：
- 五个方法都能清楚呈现
- 方法模块之间视觉风格统一
- 用户能理解每个方法适合解决什么问题

### 任务 2.4：实现案例分析区块

目标：
- 用真实感较强的场景帮助用户理解方法如何落地

交付物：
- 案例列表或案例卡片
- 案例详情展示结构

验收标准：
- 至少展示 4 个案例
- 每个案例都包含固定结构字段
- 用户可以清楚看到从问题到判断的分析路径

### 任务 2.5：实现外部资料区块

目标：
- 提供可靠的延伸学习入口

交付物：
- 外部资料卡片列表
- 来源、摘要、分类标签、链接展示

验收标准：
- 每条资料都有来源标识
- 资料可以按用途分类呈现
- 第三方内容只展示摘要和链接，不复制长段原文

## 4. 阶段三：交互与学习体验增强

### 阶段目标

补充 P1 功能，让页面从静态阅读变成可探索、可练习的学习体验。

### 任务 3.1：实现页面内锚点导航增强

目标：
- 提升长页面浏览效率

交付物：
- 当前阅读位置高亮
- 快速跳转到“概念 / 方法 / 案例 / 练习”的导航能力

验收标准：
- 滚动时当前模块状态准确
- 点击导航跳转位置准确
- 移动端导航不遮挡主要内容

### 任务 3.2：实现动态展开与对照视图

目标：
- 降低信息密度，提高比较和理解效率

交付物：
- 卡片展开查看详情
- “正确理解 vs 常见误解”对照面板
- 方法步骤的分步展示

验收标准：
- 展开和收起状态清晰
- 动态内容不会造成明显布局跳动
- 对照内容能帮助用户快速判断差异

### 任务 3.3：实现名人名言模块

目标：
- 用引用内容补充思考角度和页面节奏

交付物：
- 名人名言列表、轮播或切换视图
- 按主题筛选的标签
- 作者和出处信息

验收标准：
- 每条名言都显示作者与出处状态
- 无法确认出处的内容明确标注“待确认”
- 名言模块不干扰主学习路径

### 任务 3.4：实现实践练习模块

目标：
- 让用户通过轻量练习应用批判性思维方法

交付物：
- 练习题卡片
- 用户判断选项
- 回答后的解析反馈

验收标准：
- 至少包含 3 道练习题
- 每道题都提供解析或思考提示
- 反馈内容能对应核心方法模块

## 5. 阶段四：扩展能力与验收

### 阶段目标

完善首版质量，明确后续扩展方向，并对页面进行整体验收。

### 任务 4.1：整理 P2 扩展方案

目标：
- 将暂不实现的能力转化为后续可评估任务

交付物：
- 自动抓取外部资料方案说明
- 个性化学习路径设想
- 收藏、进度记录、关联推荐的功能草案

验收标准：
- 每个扩展功能都有目标、价值和依赖说明
- 明确哪些功能需要后端、登录或第三方接口
- 不影响首版交付范围

### 任务 4.2：进行内容与引用检查

目标：
- 确保页面内容准确、来源清晰、版权风险可控

交付物：
- 内容检查清单
- 引用来源检查结果
- 待确认问题列表

验收标准：
- 外部资料都包含来源信息
- 不展示未经确认的大段引用
- 待确认内容集中记录，便于后续处理

### 任务 4.3：进行响应式与可用性检查

目标：
- 确保页面在常见设备上可读、可用、无明显布局问题

交付物：
- 桌面端检查结果
- 移动端检查结果
- 问题修复清单

验收标准：
- 页面主要内容在桌面端和移动端都能正常阅读
- 导航、卡片、练习题等交互不会遮挡内容
- 文字、按钮和标签不发生明显溢出

### 任务 4.4：完成首版验收

目标：
- 判断项目是否达到首版可交付状态

交付物：
- 首版验收记录
- 未完成项列表
- 下一阶段建议

验收标准：
- P0 功能全部完成
- P1 核心体验基本完成
- 已知风险和待确认事项有记录
- 页面能支撑完整学习路径

## 6. 阶段依赖关系

- 阶段一是所有后续工作的基础，必须先完成核心内容和信息结构。
- 阶段二依赖阶段一的内容输入，重点完成 P0 页面展示。
- 阶段三依赖阶段二的页面结构，重点增强交互和学习体验。
- 阶段四依赖前三个阶段的成果，负责质量检查、扩展规划和最终验收。

## 7. 推荐执行顺序

1. 完成核心概念、方法、误区、案例、资料的内容整理
2. 搭建页面整体结构和导航
3. 实现 P0 内容模块
4. 增加锚点导航、展开、对照等 P1 交互
5. 增加名言与实践练习模块
6. 检查外部资料来源与引用风险
7. 完成响应式和可用性检查
8. 整理 P2 扩展方案与首版验收记录

## 8. 风险与待确认事项

### 风险

- 内容过多导致页面阅读负担过重
- 外部资料来源质量不稳定
- 名人名言出处难以完全确认
- 交互设计过多可能干扰学习节奏
- 自动抓取外部资料会引入版权、稳定性和审核问题

### 待确认事项

- 是否采用单页结构作为首版最终方案
- 是否需要中英双语
- 是否需要登录、收藏和学习进度
- 是否接入自动搜索或外部内容抓取
- 是否需要生成式内容辅助案例解释
- 首版是否需要真实上线，还是只作为本地演示项目
