# 图书投票应用 MVP 任务管理计划

## 1. 项目概览

| 项目 | 内容 |
| --- | --- |
| 项目名称 | 图书投票应用 |
| MVP 目标 | 基础 MVP、注册登录、管理员管理、书目自动补齐、本地真实封面绑定、Apple 风格列表视图、投票占比进度条、排行榜 Top 3、每人最多三票、管理员重置票数、顶部结构优化、新建图书自动抓取和邮箱密码找回均已完成 |
| 技术方向 | 前端使用 HTML/CSS/JavaScript，后端使用 Node.js 原生 HTTP 服务 + 唯一第三方依赖 `nodemailer`（仅用于 SMTP 发件），数据使用本地 JSON 文件存储 |
| 本地访问地址 | 启动 `npm start` 后访问 `http://127.0.0.1:3002`；默认端口已避开本机其他项目占用的 `3000`、`3001` |
| 局域网分享地址 | `http://192.168.3.100:3002`（同一 WiFi 下其他设备访问；IP 若变，终端跑 `ipconfig getifaddr en0` 查看当前 IP 并更新 `.env` 中 `APP_BASE_URL`）|
| 当前阶段 | 页面实际包含 8 本候选书、排行榜 Top 3、投票占比、剩余票数、管理员图书管理（含新建图书 + 豆瓣自动抓取 + 删除）、一键重置票数、邮箱注册和密码找回；通过 `192.168.3.100:3002` 可在同一 WiFi 下分享给其他设备 |
| 规格文档 | `spec.md` |

## 3. 开发阶段划分

1. 项目初始化：建立 Node.js 项目结构、运行脚本和基础目录。
2. 数据文件设计：创建并验证 `book.json` 和 `vote.json` 的初始结构。
3. 后端接口开发：实现图书列表、投票结果和提交投票接口。
4. 前端页面开发：实现图书展示、投票按钮、票数展示和状态提示。
5. 前后端联调：打通页面初始化、投票提交和票数刷新流程。
6. 异常处理：覆盖参数错误、图书不存在、文件异常和网络请求失败。
7. 测试与验收：按 MVP 验收标准验证核心流程。
8. 用户模块扩展：增加基础注册、登录、退出登录、会话和投票权限控制。
9. 管理员图书管理：增加管理员角色和图书内容编辑能力。
10. 图书数据扩展：在原 5 本书基础上新增 3 本，候选图书总数达到 8 本。
11. 书目信息补齐：管理员输入书名后，自动补齐作者、简介和封面地址建议。
12. 真实封面维护：将候选图书封面替换为对应真实书籍封面。
13. 书籍目录列表化：将候选图书展示从画廊式或网格卡片排列调整为现代 SaaS 风格纵向列表视图。
14. 投票占比进度条：在每本书票数区域展示该书票数占总票数的百分比和进度条。
15. 阶段二竞赛展示效果增强：增加排行榜 Top 3、前三名标签、最高票提示强化和 0 票空状态优化。
16. 阶段三竞赛体验优化：每人最多三票、标题和金句更新、Apple 风格统一、管理员一键重置票数、管理员列表化和顶部结构重整。
17. 后续扩展规划：记录数据库、防重复投票、投票时间窗口等后续方向。

## 16. 后续维护规则

- 每完成一个任务，需要将任务状态从 `未开始` 或 `进行中` 更新为 `已完成`。
- 每开始一个任务，需要将任务状态更新为 `进行中`。
- 如果任务无法继续，需要将任务状态更新为 `阻塞`，并在“项目进度同步区”记录阻塞原因。
- 每次开发结束后，需要同步“当前整体进度”“最近完成的任务”“当前正在进行的任务”“下一步计划”和“阻塞问题”。
- 如果需求发生变化，需要同时更新 `spec.md` 和 `task.md`，保证规格说明和任务计划一致。
- MVP 验收清单应在测试与验收阶段逐项勾选，不应提前标记完成。
