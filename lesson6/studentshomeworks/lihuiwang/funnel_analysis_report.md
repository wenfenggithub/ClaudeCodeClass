## 第二步：转化率计算

### Claude Prompt

```
读取 funnel_data.csv（包含 visitors, registrations, trials, payments, renewals 五列）。
计算每层转化率（相对上一层）、整体转化率、各阶段流失率。输出格式化表格。
```

### Python 代码

```python
import pandas as pd

df = pd.read_csv('funnel_data.csv')
stages = ['visitors','registrations','trials','payments','renewals']
labels = ['访问首页', '注册账号', '开始试用', '完成付费', '续费']
values = [df[s].sum() for s in stages]

# 计算各层转化率
print(f"{'阶段':<12} {'用户数':>8} {'转化率':>10} {'流失率':>10}")
print("-" * 45)
for i in range(len(stages)):
    if i == 0:
        print(f"{labels[i]:<10} {values[i]:>10,}")
    else:
        rate = values[i] / values[i-1] * 100
        drop = 100 - rate
        print(f"{labels[i]:<10} {values[i]:>10,} {rate:>8.1f}% {drop:>9.1f}%")

# 整体转化率
total_rate = values[-1] / values[0] * 100
print(f"\n整体转化率（访问→续费）: {total_rate:.2f}%")
```

## 第三步：问题环节定位

### 🔴 最大流失点：访问首页 → 注册账号

- **流失人数**：6,500 人（占比 65%）
- **转化率仅**：35.0%，远低于 SaaS 行业健康值（60%+）

### 三维度故障分析

| 维度 | 可能原因 |
|------|----------|
| **产品** | 注册表单字段过多（如>3个必填），缺少社交账号快捷登录；首页缺乏产品价值展示，用户看不到"注册后能得到什么" |
| **运营** | 未设置注册引导奖励（如免费试用天数放大）；缺乏紧迫感（限时优惠、限量名额）；注册流程中未展示 Social Proof |
| **体验** | 页面加载慢导致跳出；移动端适配差；注册验证码体验差；无进度条 |

### 🟡 次大流失点：注册 → 试用（65.7%）

- 注册完成后缺乏引导 → 用户"不知道下一步干嘛"
- 可能需要在注册成功后弹窗引导、邮件 Onboarding 序列、首次登录的交互式引导

### 🟢 表现最好：续费（62.2%）

- 付费用户中超过 60% 会续费，产品核心价值得到认可
- 若进一步提升至 70%，对 LTV 有显著贡献

### 🔍 洞察 1：当前漏斗最大的瓶颈是什么？

**访问 → 注册**。65% 的访客在注册前流失，这是最大的单一优化机会。

**优先优化「访问→注册」**。

### 📈 洞察 3：提升空间预估

如果把四个环节的转化率各提升 5%：

| 环节 | 当前转化率 | 目标转化率 | 付费用户变化 |
|------|-----------|-----------|-------------|
| 访问→注册 | 35.0% | 40.0% | +14.3% |
| 注册→试用 | 34.3% | 39.3% | +14.6% |
| 试用→付费 | 37.5% | 42.5% | +13.3% |
| 付费→续费 | 62.2% | 67.2% | +8.0% |

**每环节优化 5% 的叠加效应可使总付费用户增长约 60%。**

### 💡 洞察 4：对业务增长的启发

1. **漏斗顶部决定天花板**：访问→注册是最该砸资源的地方，1 个百分点的提升比其他环节影响力大得多
2. **试用→付费是价值验证点**：37.5% 偏低，说明试用体验不足以让用户感知"付了费能得到什么"
3. **续费率是护城河**：62.2% 虽不错，但 SaaS 优秀标准是 70-80%。续费不靠促销，靠产品力
4. **不要平均用力**：资源应该 60% 投在流量→注册，25% 投在试用→付费，15% 投在续费
