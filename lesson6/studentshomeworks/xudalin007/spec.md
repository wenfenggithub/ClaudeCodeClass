# 第 6 课作业规格说明：漏斗分析自学与案例项目

## 一、作业主题

用 AI 自学数据分析中的「漏斗分析」模块，并完成一份适合个人学习场景的案例教学 PPT 和一个案例项目。

## 二、学习者背景

- 学习者是成年人。
- 工作背景：装修公司分公司经理，主要负责经营和管理。
- AI 工具使用经验：有基础经验。
- 数据分析基础：一般，适合初学者内容。
- 内容风格要求：通俗易懂、贴近经营管理场景，不要过于学术化。

## 三、作业目标

1. 用 AI 自学「漏斗分析」这个数据分析模块。
2. 用 AI 制作一份适合个人自学的案例教学 PPT，HTML 格式即可。
3. 优化教学 PPT，使学习者可以通过 PPT 掌握漏斗分析。
4. 根据 PPT 中的一个案例，完成一个完整案例项目。
5. 最终提交两个成果文件：
   - `funnel_analysis_lesson.html`
   - `decoration_funnel_case_project.html`

## 四、主要案例方向

案例主题：装修公司客户转化漏斗分析。

案例应围绕装修客户从广告曝光到最终签约的全过程展开。

建议漏斗步骤：

1. 广告曝光
2. 留下联系方式
3. 客服有效沟通
4. 预约量房
5. 实际上门量房
6. 出设计方案
7. 报价沟通
8. 最终签约

## 五、第一部分任务：设计教学 PPT 大纲

先生成 12-15 页左右的 HTML PPT 大纲。

每一页都要包括以下内容：

1. 页面标题
2. 核心内容
3. 是否需要图表
4. 讲解备注

大纲需要由浅入深，适合数据分析初学者自学。

## 六、第二部分任务：生成完整 HTML 教学 PPT

根据 PPT 大纲，生成一个可以直接在浏览器打开的 HTML PPT 文件。

文件名：

`funnel_analysis_lesson.html`

### HTML PPT 形式要求

1. PPT 采用 16:9 横版页面，适合电脑浏览器查看和投影展示。
2. 每一页作为独立幻灯片展示，支持上一页、下一页按钮。
3. 支持键盘左右方向键切换页面。
4. 页面底部显示当前页码和总页数。
5. 内容排版要留有空白，避免一页中堆放过多文字。
6. 不依赖外部图表库或联网资源，文件离线也能打开。

### 教学 PPT 内容要求

1. 风格简洁、清晰，适合成年人自学。
2. 每页内容不要太拥挤。
3. 重点讲清楚以下概念：
   - 什么是漏斗分析
   - 转化率
   - 流失率
   - 整体转化率
4. 必须包含装修公司客户转化漏斗案例。
5. 必须包含：
   - 数据表
   - 公式
   - 漏斗图
   - 分析结论
   - 优化建议
6. HTML 文件可以直接复制保存后打开使用。
7. 页面内容使用中文。
8. 不依赖复杂外部库，尽量使用 HTML、CSS、JavaScript 实现。

### 计算口径要求

1. 单步转化率 = 当前步骤人数 / 上一步人数。
2. 单步流失率 = 1 - 单步转化率。
3. 单步流失人数 = 上一步人数 - 当前步骤人数。
4. 整体转化率 = 最终签约人数 / 广告曝光人数。
5. 百分比结果统一保留 1 位小数。
6. 需要同时指出：
   - 流失人数最多的环节
   - 流失率最高的环节
   - 最值得优先优化的环节及理由

## 七、第三部分任务：生成案例项目

基于「装修公司客户转化漏斗分析」生成一个完整案例项目，HTML 格式。

文件名：

`decoration_funnel_case_project.html`

### 案例项目必须包括

1. 项目标题
2. 项目背景
3. 模拟数据表
4. 每一步转化率计算
5. 每一步流失率计算
6. 整体转化率计算
7. 漏斗图可视化
8. 关键问题发现
9. 业务原因分析
10. 优化建议
11. 项目总结
12. 流失人数最多和流失率最高环节的对比分析
13. 结合经营管理视角说明优先优化哪个环节

### 建议模拟数据

| 漏斗步骤 | 人数 |
|---|---:|
| 广告曝光 | 10000 |
| 留下联系方式 | 800 |
| 客服有效沟通 | 500 |
| 预约量房 | 260 |
| 实际上门量房 | 180 |
| 出设计方案 | 120 |
| 报价沟通 | 90 |
| 最终签约 | 36 |

### 经营指标补充

案例项目中可以加入以下经营假设，帮助学习者理解漏斗分析对经营决策的价值：

1. 本轮广告投放成本：30000 元。
2. 平均每个签约客户合同额：120000 元。
3. 获客成本 = 广告投放成本 / 最终签约人数。
4. 签约收入 = 最终签约人数 * 平均合同额。
5. 可以模拟某个关键环节转化率提升后，多带来的签约人数和签约收入。

这些经营指标不要求过于复杂，重点是说明：漏斗分析不仅能发现问题，还能帮助判断哪个环节最值得投入资源优化。

### 漏斗图要求

1. 漏斗图可使用 HTML、CSS、SVG 或 Canvas 实现。
2. 不使用外部图表库。
3. 漏斗图中应展示每一步的人数。
4. 建议同时展示每一步相对上一环节的转化率和流失率。
5. 漏斗宽度应随人数减少而逐步变窄，使初学者能直观看出流失过程。

## 八、第四部分任务：输出要求

最终需要分别输出两个完整 HTML 文件内容：

1. `funnel_analysis_lesson.html`
2. `decoration_funnel_case_project.html`

每个 HTML 文件都必须结构完整，包括：

```html
<!DOCTYPE html>
<html>
<head>
<meta charset="UTF-8">
<title>标题</title>
<style>页面样式</style>
</head>
<body>页面内容</body>
</html>
```

## 九、第五部分任务：质量要求

1. 内容要由浅入深，适合初学者。
2. 不要只讲概念，必须结合装修公司案例。
3. 不要只生成文字，要有表格、公式、图表和结论。
4. 重点突出：通过漏斗分析找到客户流失最多的环节，并提出经营改善建议。
5. 最终成果要像一份可以提交的作业，而不是普通聊天回答。

## 十、最终验收标准

1. 两个 HTML 文件都可以直接双击打开，不需要安装额外依赖。
2. 两个 HTML 文件均为中文内容。
3. `funnel_analysis_lesson.html` 包含 12-15 页教学 PPT。
4. `decoration_funnel_case_project.html` 包含完整案例项目结构。
5. 两个文件都包含数据表、公式、漏斗图、分析结论和优化建议。
6. 所有转化率、流失率、整体转化率的计算结果与模拟数据一致。
7. 明确指出流失人数最多的环节、流失率最高的环节，以及优先优化建议。
8. 页面在普通电脑屏幕上阅读不拥挤，按钮、文字和图表不重叠。
9. 内容贴近装修公司经营管理场景，避免过度学术化表达。

## 十一、建议生成顺序

1. 先完成教学 PPT 大纲。
2. 根据大纲生成 `funnel_analysis_lesson.html`。
3. 从教学 PPT 案例中延伸出完整项目。
4. 生成 `decoration_funnel_case_project.html`。
5. 检查两个 HTML 文件是否可以直接在浏览器打开。
6. 检查是否包含表格、公式、漏斗图、结论和优化建议。
