# AI 数据分析与可视化 · 结构化知识地图

> 面向：非技术背景成年人，有基本 AI 工具使用经验，目标是「能用 AI 辅助完成日常数据分析与可视化」
> 一句话：**数据分析是从数据里找答案，数据可视化是把答案讲清楚；AI 让这两件事的门槛大幅降低。**

---

## 第一层 · 定位（这个模块是什么）

### 1. 它在整个领域中的位置

数据科学是一个大领域，从「拿到数据」到「产生价值」是一条链路：

```
数据采集 → 数据工程 → 【数据分析】 → 【数据可视化】 → 决策/行动
 (爬取/埋点)  (清洗/存储)   (找规律/答问题)  (讲清楚/做图表)
                                  ↑ 你在这里 ↑
                         机器学习/AI建模 是它的「进阶分支」
```

| 层级 | 做什么 | 是否本模块 |
|------|--------|-----------|
| 数据工程 | 把数据搬运、清洗、存好 | 相邻（上游） |
| **数据分析** | **从数据里找规律、回答业务问题** | ✅ 核心 |
| **数据可视化** | **把分析结果用图表讲清楚** | ✅ 核心 |
| 机器学习/AI 建模 | 用数据做预测、分类 | 相邻（进阶） |
| BI 报表 | 把指标做成固定看板、定期刷新 | 相邻（产品化） |

**AI 的角色：** 过去这些工作需要写 SQL、Python、统计知识。现在 AI（如 Claude）可以帮你写代码、清洗数据、跑分析、生成图表，你的核心能力从「会写代码」转向「会提问、会判断结果」。

### 2. 主要解决什么问题

| ✅ 它解决 | ❌ 它不解决 |
|----------|------------|
| 这个月销售为什么下滑？ | 数据本身怎么采集（那是数据工程） |
| 哪类客户最容易流失？ | 预测明年精确销量（那偏机器学习） |
| 两个产品哪个表现更好？ | 决策本身该怎么做（数据只提供依据） |
| 怎么把结论一张图讲清楚？ | 保证数据 100% 真实（需业务核对） |

一句话：**它把「一堆看不懂的数据」变成「一个能行动的结论」。**

### 3. 与相邻模块的区别

| 概念 | 一句话定义 | 关键区别 |
|------|-----------|---------|
| 数据分析 | 从数据中找答案 | 探索性、回答「为什么/是什么」 |
| 数据工程 | 把数据准备好 | 偏工程，处理量大、管道化 |
| 机器学习 | 用数据做预测 | 偏未来、自动化决策 |
| BI / 报表 | 固定指标的持续监控 | 重复性、看板化、给管理层看 |
| 数据可视化 | 把信息用图表表达 | 是分析的「表达环节」，也可独立成设计 |

> 新手常见误区：把「做图表」当成数据分析的全部。其实**图表只是最后一步，前面的「问对问题 + 清洗 + 找规律」才是核心**。

## 第二层 · 知识结构（要学什么）

### 4. 前置知识

| 必须先掌握（边做边补即可，不用系统学） | 可以完全交给 AI / 后补 |
|------|------|
| 表格的基本概念：行=记录、列=字段 | 编程语法（Python/SQL，让 AI 写） |
| 基础数学直觉：平均数、占比、增长率 | 复杂统计推断（假设检验等） |
| 会用 Excel / Google Sheets 看数据 | 数据库原理 |
| 能清晰描述你的业务问题 | 可视化库 API（matplotlib 等） |

**核心心态：** 你不需要先学完编程才能开始。先有一个真实问题和一份数据，剩下的边做边问 AI。

### 5. 核心概念（10–15 个关键术语）

| 术语 | 大白话解释 | 生活化例子 |
|------|-----------|-----------|
| 数据集 (Dataset) | 一张或多张表格 | 一份 Excel 销售明细 |
| 字段 / 变量 (Column) | 表格的一列 | 「销售额」「日期」「城市」 |
| 记录 (Row) | 表格的一行 | 一笔订单 |
| 数据清洗 (Cleaning) | 把脏数据修好 | 删重复、补空值、统一格式 |
| 缺失值 (Missing Value) | 该有却空着的格子 | 有的订单没填城市 |
| EDA（探索性分析） | 先随便看看数据长啥样 | 先翻翻账本找感觉 |
| 描述统计 | 用几个数字概括一堆数据 | 平均客单价、最大单笔 |
| 分组聚合 (Group By) | 按某列分类后算汇总 | 按城市看各自总销售 |
| 相关性 (Correlation) | 两个变量一起变的程度 | 广告投入↑ 销售↑ |
| 趋势 (Trend) | 随时间的变化方向 | 月销售逐月上升 |
| 异常值 (Outlier) | 明显不合群的数据点 | 某天销售是平时 50 倍 |
| 维度 vs 指标 | 维度=分类角度，指标=被衡量的数 | 维度「城市」，指标「销售额」 |
| 数据透视表 (Pivot) | 多角度交叉汇总 | Excel 透视表 |
| Dashboard（仪表盘） | 多个图表组成的监控面板 | 老板看的经营大屏 |
| 数据故事 (Data Storytelling) | 用数据讲一个有逻辑的故事 | 「下滑→原因→建议」三段式 |

### 6. 核心方法与工具

**标准分析流程（记住这 6 步就够）：**

```
1. 问对问题  →  2. 取数+清洗  →  3. 探索(EDA)  →  4. 找规律  →  5. 可视化  →  6. 讲结论
```

**常见可视化类型（选图口诀：看什么 → 用什么图）：**

| 你想表达 | 推荐图表 | 例子 |
|----------|---------|------|
| 随时间变化 | 折线图 | 月销售趋势 |
| 比较大小 | 柱状图 / 条形图 | 各城市销售对比 |
| 占比构成 | 饼图 / 堆叠柱 | 各产品销售占比 |
| 两个变量关系 | 散点图 | 广告投入 vs 销售 |
| 分布情况 | 直方图 / 箱线图 | 客单价分布 |
| 多指标对比 | 雷达图 / 热力图 | 区域×品类表现 |

**常用工具（标注 AI 辅助程度）：**

| 工具 | 用途 | AI 辅助程度 |
|------|------|------------|
| Excel / Google Sheets | 看数据、简单透视 | ⭐⭐ 入门首选 |
| Claude / ChatGPT | 直接传表格让 AI 分析+画图 | ⭐⭐⭐⭐⭐ 最推荐起点 |
| Python (pandas/matplotlib) | 灵活强大，AI 代写代码 | ⭐⭐⭐⭐ |
| SQL | 从数据库取数 | ⭐⭐⭐⭐ AI 代写 |
| BI 工具 (Tableau/Power BI/Metabase) | 做交互式看板 | ⭐⭐⭐ |

> 给你的建议路径：**Excel 看数据 → Claude 做分析 → 需要更强时让 AI 写 Python**。

## 第三层 · 应用与能力（学了能干什么）

### 7. 常见应用场景

1. **业务复盘**：分析季度销售，找出增长点和流失点，给管理层做汇报。
2. **运营优化**：分析用户行为，找出哪个环节流失最多。
3. **市场分析**：对比不同渠道 / 产品 / 区域的投入产出比。
4. **个人/投资**：分析定投收益、记账数据、健康数据。
5. **报告自动化**：把每月例行报表交给 AI 半自动生成。

### 8. 能力分级

| 能力维度 | 入门（听过会看） | 能用（能独立做） | 熟练（有判断力） |
|----------|-----------------|-----------------|-----------------|
| 提问 | 能说出模糊需求 | 能拆成可分析的具体问题 | 能定义指标和口径 |
| 清洗 | 知道有脏数据 | 能让 AI 清洗并检查 | 能识别数据陷阱 |
| 分析 | 看得懂图表 | 能做分组、对比、趋势 | 能区分相关与因果 |
| 可视化 | 会插入图表 | 能选对图、讲清楚 | 能做数据故事/看板 |
| AI 协作 | 会让 AI 出结果 | 会验证 AI 结果对错 | 能设计分析 Prompt 流程 |

### 9. 学习路径与核心 20%

**最该优先掌握的「核心 20%」（掌握它能解决 80% 的问题）：**

1. ✅ **问对问题** —— 把「帮我分析下销售」变成「对比 Q1/Q2 各城市销售增长率，找出下滑最多的城市」
2. ✅ **分组聚合（Group By）** —— 90% 的业务分析都是「按某个维度算某个指标」
3. ✅ **趋势 + 对比两类图** —— 折线（趋势）和柱状（对比）覆盖绝大多数场景
4. ✅ **相关 ≠ 因果** —— 避免最常见的分析错误
5. ✅ **让 AI 做、自己验证** —— 学会检查 AI 给的数字对不对

**由浅入深的学习顺序：**

```
第 1 周：用 Claude 分析一份真实 Excel 表（提问→AI 出结论+图）
第 2 周：学分组聚合 + 选对图表类型（趋势/对比/占比）
第 3 周：学数据清洗 + 识别异常值、缺失值
第 4 周：做一个完整「数据故事」汇报（下滑→原因→建议）
进阶：让 AI 写 Python 处理更大数据 / 学做交互式 Dashboard
```

## 推荐的第一个练习（最小闭环）

> 拿一份真实的销售/记账/运营表格，把下面这段发给 Claude：

```
这是我的一份【销售明细】表格（附件）。请帮我：
1. 先描述这份数据有几行几列、每列是什么、有没有缺失或异常
2. 按【城市】分组，算出各城市的总销售额和占比，用柱状图展示
3. 按【月份】看整体销售趋势，用折线图展示，指出增长或下滑
4. 给我 3 条基于数据的业务结论和建议
5. 最后告诉我：这份分析里有哪些地方我需要人工核对、不能全信 AI
```

跑完这一遍，你就完成了从「问问题」到「讲结论」的完整闭环。
