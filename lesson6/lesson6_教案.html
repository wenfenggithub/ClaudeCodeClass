案例详解

案例 1 入门 · 读数据 ⏱ 15 分钟

看懂一份数据 —— 让 Claude 当你的"数据翻译官"

场景

你记了三个月的家庭账本，存成一个 CSV 文件，但密密麻麻看不出名堂。第一步不是分析，而是先让 Claude 告诉你：这份数据里到底有什么。

示例数据 家庭记账.csv
日期类别项目金额支付方式
2026-03-01餐饮早餐豆浆油条12微信
2026-03-01交通地铁6交通卡
2026-03-02购物超市日用品238支付宝
2026-03-03餐饮家庭聚餐420信用卡
2026-03-05居住电费186银行卡
…（共约 120 行，覆盖 3 月 ~ 5 月）

提示学员：课堂上可让 Claude 帮忙生成这份示例数据（见下方提示词最后一句）。

照抄给 Claude 的提示词
PROMPT我有一个家庭记账的 CSV 文件 家庭记账.csv。 请你读取它，然后用通俗的语言告诉我： 1. 这份数据一共有多少行、多少列，每一列分别是什么意思； 2. 时间范围是从哪天到哪天； 3. 有没有明显的空白、重复或异常的数据； 4. 用三句话总结这份账本的整体情况。 如果文件还不存在，请先帮我造一份约 120 行、覆盖 3~5 月、 包含餐饮/交通/购物/居住/娱乐等类别的示例数据，再做分析。
预期产出

Claude 会读取文件（或先生成），输出一段"数据体检报告"：行列数、字段含义、时间跨度、数据质量提示，以及一段大白话总结。这一步不产生图表，只产生"理解"。

老师讲解要点
核心心法：分析任何数据前，先让 Claude 做"数据体检"。这一步能避免后面在错误的数据上白忙活。强调"让 Claude 帮忙造演示数据"这个技巧——后面每个案例都能用。
↓会读了，下一步就是算——把数字变成有用的统计结果。
案例 2 入门 · 算统计 ⏱ 15 分钟

简单统计与排名 —— 一句话搞定 Excel 公式的活儿

场景

班主任手里有一张期中成绩单，想知道：班级平均分、谁是前三名、哪门课最拉胯。以前要拉公式、做排序，现在直接说人话。

示例数据 成绩单.csv
学号姓名语文数学英语
01张伟889276
02李娜957890
03王芳728568
04刘强806082
05陈静919488
…（共 30 名学生）
照抄给 Claude 的提示词
PROMPT用 成绩单.csv（30 名学生，含语文/数学/英语三门课）做以下分析： 1. 给每个学生算出总分和平均分，并加一列"总分排名"； 2. 算出三门课各自的班级平均分、最高分、最低分； 3. 告诉我总分前三名是谁，以及哪一门课全班整体最弱； 4. 把带排名的完整结果另存为 成绩单_结果.csv。 请一步步说明你做了什么，结果用表格展示给我看。
预期产出

一段 Python 脚本（Claude 自动写好并运行）+ 排序后的成绩表 + 一份新文件 成绩单_结果.csv + 一句话结论（"全班英语最弱，平均 78 分"）。

老师讲解要点
  • 强调"提需求"而不是"写代码"：你只说要什么，Claude 负责怎么做。
  • 引导学员要求 Claude 把结果存成新文件，养成"保留产物"的习惯，为后续案例铺路。
↓真实数据没这么干净。下一步学清洗——把脏数据收拾整齐。
案例 3 进阶 · 清洗数据 ⏱ 20 分钟

数据清洗 —— 收拾一张"脏乱差"的订单表

场景

从系统导出的电商订单表，日期格式五花八门、金额带着"￥"和空格、还有重复行和空值。这是真实工作中最常见、也最费时的环节。

示例数据 订单_脏数据.csv
订单号客户下单日期金额状态
1001张伟2026/3/1￥ 238.00已完成
1002李娜2026-03-02420已完成
1003（空）3月3日￥186已取消
1002李娜2026-03-02420已完成
1004王芳2026.3.5￥ 99.9 完成
注意：日期 4 种写法、金额带符号、1002 重复、客户有空值、状态"已完成/完成"不统一
照抄给 Claude 的提示词
PROMPT订单_脏数据.csv 是从系统导出的脏数据，请帮我清洗： 1. 把"下单日期"统一成 YYYY-MM-DD 标准格式； 2. 把"金额"清理成纯数字（去掉 ￥ 符号和空格），列类型转为数值； 3. 删除完全重复的订单行； 4. "状态"里"完成"和"已完成"统一成"已完成"； 5. 客户为空的行，单独列出来给我看，先不要删。 清洗前后各告诉我有多少行，列出你做的每一处修改， 并把干净数据存为 订单_清洗后.csv。
预期产出

一份"清洗日志"（改了哪些、删了几行重复、几个空值）+ 规整后的 订单_清洗后.csv + 一张需要人工确认的"问题行"清单。

老师讲解要点
关键提醒：清洗时务必让 Claude列出每一处改动、并对"删除"类操作保持谨慎（空值先列出、不要急着删）。数据清洗是"可逆、可核对"的，不能黑箱操作。这是本节课最重要的职业习惯。
↓单张表干净了，真实分析往往要多张表一起看。
案例 4 进阶 · 多表合并 ⏱ 20 分钟

多表合并 —— 把"订单表"和"客户表"拼到一起

场景

订单表里只有客户名，但客户的城市、会员等级在另一张表里。想分析"哪个城市的金牌会员买得最多"，就得先把两张表关联起来。

示例数据（两张表）

订单_清洗后.csv（沿用案例3）

订单号客户金额
1001张伟238.0
1002李娜420.0
1004王芳99.9

客户信息.csv

客户城市会员等级
张伟北京金牌
李娜上海普通
王芳北京金牌
照抄给 Claude 的提示词
PROMPT我有两张表：订单_清洗后.csv（含 客户、金额）和 客户信息.csv（含 客户、城市、会员等级）。 1. 请用"客户"这一列把两张表合并成一张完整的表； 2. 如果有订单的客户在客户表里找不到，单独告诉我（不要丢掉）； 3. 合并后，按"城市 + 会员等级"汇总各自的总消费金额； 4. 结果存为 订单_含客户信息.csv，并把汇总表展示给我。
预期产出

一张合并后的宽表 + 一份"城市 × 会员等级"的消费汇总（如：北京金牌 337.9 元）+ 一份匹配不上的订单清单。

老师讲解要点
  • 用大白话解释"合并/关联"：就是用一个共同的列（这里是"客户"）把两张表对上号，像拼图。
  • 强调"匹配不上的要单独看"——合并最容易悄悄丢数据，必须让 Claude 报告。
↓数据整理到位了，现在让它开口说话——画成图。
案例 5 可视化 · 基础 ⏱ 20 分钟

基础可视化 —— 把月度销售画成趋势图

场景

一串销售数字看不出感觉，画成折线图后，"哪个月涨了、哪个月跌了"一眼就懂。这是数据从"看不懂"到"有洞察"的关键一跳。

示例数据 月度销售.csv
月份线上销售额线下销售额
2026-015200048000
2026-026100045000
2026-035800052000
2026-047300050000
2026-058900047000
2026-069500049000
照抄给 Claude 的提示词
PROMPT用 月度销售.csv 画一张折线图： - 横轴是月份，画出"线上"和"线下"两条销售额曲线； - 给图加上中文标题"2026 上半年线上 vs 线下销售趋势"、坐标轴标签和图例； - 数据点上标出数值，图片清晰、配色淡雅； - 把图保存成 销售趋势.png。 画完后用两三句话告诉我这张图里最值得注意的趋势。
预期产出

一个 销售趋势.png 图片文件 + 一段文字洞察（"线上从 1 月到 6 月几乎翻倍，线下基本持平，增长全靠线上"）。

老师讲解要点
常见坑：matplotlib 默认显示不了中文（会变成方框）。提前告诉学员：如果图里中文是乱码，就对 Claude 说"图里中文显示成方框了，请设置中文字体修复"，它会自动处理。
↓一张图够用了吗？真实分析常要多个角度同时看。
案例 6 可视化 · 进阶 ⏱ 20 分钟

进阶可视化 —— 运动手环数据的多维仪表盘

场景

你的运动手环导出了一个月的步数、睡眠、心率数据。单看哪一项都不够，把它们组合成一张"多图面板"，才能看出生活规律。

示例数据 手环数据.csv
日期星期步数睡眠小时静息心率
2026-05-01周五82007.562
2026-05-02周六125008.258
2026-05-03周日31009.060
2026-05-04周一68006.365
2026-05-05周二74006.864
…（共 31 天）
照抄给 Claude 的提示词
PROMPT用 手环数据.csv（31 天的步数、睡眠、心率）做一张组合图， 分成上下三个子图，共用一个横轴（日期）： 1. 上：每日步数柱状图，周末用不同颜色突出； 2. 中：每日睡眠小时折线图，画一条 7 小时的参考虚线； 3. 下：静息心率折线图。 整体配色淡雅、标题清晰，保存为 健康面板.png。 最后帮我分析：我的步数、睡眠在工作日和周末有什么规律？ 睡得少的那几天，第二天心率有没有变高？
预期产出

一张三合一的 健康面板.png + 一段跨维度的规律分析（如"周末步数高、睡眠足；工作日睡不够时静息心率明显偏高"）。

老师讲解要点
  • 展示 Claude 能一次画"组合图"，并把多个指标放在一起对比。
  • 重点：让 Claude 不只是画图，还要解读图里指标之间的关系——这才是分析，不只是制图。
↓最后一步：把前面所有能力串成一份完整报告。
案例 7 综合 · 出结论 ⏱ 20 分钟

从数据到结论 —— 一句话生成完整分析报告

场景

老板说"把这季度销售情况理一份报告给我"。这一步把前面学的读、洗、合、画全部打包：让 Claude 自动完成从原始数据到一份图文报告的全流程。

示例数据 季度销售明细.csv
日期地区品类销售额客户数
2026-04-01华北家电3200014
2026-04-01华东服饰1850040
2026-04-02华南家电2700011
2026-04-02华东食品960052
…（约 300 行，覆盖 4~6 月、3 个地区、4 个品类）
照抄给 Claude 的提示词
PROMPT季度销售明细.csv 是 Q2 的销售流水（地区、品类、销售额、客户数）。 请帮我生成一份给管理层看的季度分析报告，包含： 1. 总体概览：总销售额、总客户数、客单价； 2. 按"地区"和"品类"分别汇总，各画一张柱状图； 3. 月度销售趋势折线图； 4. 找出表现最好和最差的地区/品类，并尝试给出可能的原因； 5. 3 条可执行的下季度建议。 把所有图保存为 PNG，并把报告整理成一个 季度报告.md 文件， 图文并茂，语言简洁、面向非技术的管理者。
预期产出

多张图表 PNG + 一份结构化的 季度报告.md（含概览、图表、亮点/短板、建议）。可现场再用 /make-pdf 之类的方式导出成 PDF。

老师讲解要点
升华：到这一步学员应该意识到——他们指挥的不是一个"画图工具"，而是一个"会自己读数据、自己决定怎么分析、还能写结论"的分析助手。强调"把结论写给非技术读者看"这种面向人的要求，是高质量提示词的标志。

经典数据集延伸案例 · 选讲 / 课后拓展

下面两个是数据科学入门绕不开的"明星数据集"，几乎所有教程都会用到。课堂时间充裕可现场演示，时间紧张则留作课后拓展。它们的好处是：网上随处可下载，Claude 也能直接帮你获取。

案例 8 经典 · 探索性分析 ⏱ 选讲 ~20 分钟

泰坦尼克号生存分析 —— 数据科学的"Hello World"

场景

1912 年泰坦尼克号沉船，891 名乘客有人生还有人遇难。这份数据被全世界用来入门"探索性数据分析"（EDA）：从一堆乘客信息里，找出"什么样的人更容易活下来"。

示例数据 titanic.csv
乘客ID是否生还船舱等级姓名性别年龄票价
10（遇难）3Braund, Mr. Owen男227.25
21（生还）1Cumings, Mrs. John女3871.28
31（生还）3Heikkinen, Miss. Laina女267.92
41（生还）1Futrelle, Mrs. Jacques女3553.10
50（遇难）3Allen, Mr. William男358.05
…（共 891 行；真实数据年龄列有约 20% 缺失，正好用来复习清洗）

数据可让 Claude 直接获取（seaborn 库自带），无需手动下载。

照抄给 Claude 的提示词
PROMPT请加载经典的泰坦尼克号数据集（可以用 seaborn 自带的 titanic， 或从网上下载 titanic.csv），帮我做探索性分析： 1. 先做数据体检：多少行、哪些列、年龄等列缺失多少； 2. 计算总体生还率，再分别按"性别""船舱等级"计算生还率； 3. 画图： - 男女生还率对比柱状图； - 各船舱等级生还率对比柱状图； - 生还者与遇难者的年龄分布对比（直方图）； 4. 用三五句话总结"什么样的乘客更容易生还"，并解释背后的原因。 图保存为 PNG，配色淡雅、标题用中文。
预期产出

数据体检报告 + 几张对比图（性别/船舱/年龄）+ 一段经典结论：女性、儿童、头等舱乘客生还率显著更高——印证了"妇女儿童优先"和阶层差异。

老师讲解要点
  • 这是把前面"读 + 洗 + 算 + 画 + 结论"五步用在真实经典数据上的完整复盘。
  • 强调 EDA 的思维：不是先有结论再找数据，而是带着问题去"逛"数据，让图表告诉你答案。
  • 引导学员体会"分组对比"（按性别、按等级分别算生还率）是数据分析里最常用的发现规律的手法。
↓泰坦尼克看的是"分组对比"，下一个经典数据看的是变量之间的关系。
案例 9 经典 · 关系可视化 ⏱ 选讲 ~20 分钟

鸢尾花数据集 —— 用图看清"变量之间的关系"

场景

鸢尾花（Iris）数据集记录了 3 种鸢尾花、每种 50 朵的花瓣和花萼尺寸。它被用来演示一个核心问题：仅凭几个数字测量值，能不能把不同种类区分开？这是理解"相关性"和"聚类"的最佳入门。

示例数据 iris.csv
花萼长花萼宽花瓣长花瓣宽品种
5.13.51.40.2山鸢尾 setosa
4.93.01.40.2山鸢尾 setosa
7.03.24.71.4变色鸢尾 versicolor
6.43.24.51.5变色鸢尾 versicolor
6.33.36.02.5维吉尼亚 virginica
…（共 150 行，每个品种 50 行；单位均为厘米）

同样是 seaborn 自带数据集，Claude 可一行代码加载。

照抄给 Claude 的提示词
PROMPT请加载经典的鸢尾花数据集（seaborn 自带的 iris），帮我探索 "花瓣和花萼的尺寸，能不能区分三种鸢尾花"： 1. 先按品种分组，算出四个测量值的平均值，列成表格； 2. 画一张散点图：横轴花瓣长、纵轴花瓣宽，用不同颜色区分三个品种； 3. 画一张相关性热力图，看哪些测量值彼此关系最紧密； 4. （进阶）画一张 4 个变量两两组合的散点矩阵图（pairplot），按品种上色； 5. 告诉我：哪几个特征最能把品种区分开？有没有哪两种花特别难分？ 图保存为 PNG，标题用中文，配色清爽。
预期产出

分品种均值表 + 散点图 + 相关性热力图 + 散点矩阵 + 经典结论：花瓣长/宽最能区分品种；setosa 一眼就能分开，versicolor 和 virginica 有部分重叠、最难区分。

老师讲解要点
引入两个新概念（用大白话）：相关性——两个数一起变大变小的程度（花瓣越长往往越宽）；聚类/可分性——同类的点在图上会扎堆。让学员看到：好的图能让"能不能区分"这种问题一眼有答案，这正是可视化的威力。

综合动手练习 · 20 分钟

请学员用自己的一份真实数据（或现场用 Claude 生成一份感兴趣的数据），独立走完一个最小闭环：

  • 读：让 Claude 做一次数据体检（案例 1 的套路）。
  • 算或洗：做一个统计，或清洗一处脏数据（案例 2 / 3）。
  • 画：生成至少一张图，并要求 Claude 给出一句洞察（案例 5）。
老师巡场：重点看学员的提示词写得够不够具体。最常见的问题是"指令太笼统"——引导他们补上"要什么格式、存成什么文件、给我什么结论"。

课堂总结

🔁 一条主线

读 → 算/洗 → 合 → 画 → 出结论。任何数据任务都能套进这五步。

🗣 提需求，不写代码

你负责说清"要什么、什么格式、给我什么结论"，Claude 负责怎么实现。

🔍 永远可核对

清洗和合并要让 Claude 列出每处改动、报告丢失的数据，绝不黑箱。

📂 留住产物

每一步都让结果存成新文件，方便下一步复用、也方便出错时回溯。

课后练习

  • 基础题：导出你近三个月的真实账单（银行/微信/支付宝），让 Claude 做一次案例 1 的"数据体检"并画出消费分类饼图。
  • 进阶题：找一份带脏数据的表格，完整跑一遍案例 3 的清洗流程，并把 Claude 的"清洗日志"截图记录下来。
  • 挑战题：仿照案例 7，为你关心的任意主题（个人开支、读书记录、健身数据）生成一份图文并茂的 Markdown 分析报告。
