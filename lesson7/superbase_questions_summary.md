## 1. 查看并解释当前项目

**问题 / 需求：** “查看一下当前这个项目文件，解释一下。”

**回答要点：**
- 这是一个 **图书投票（Book Vote）Web 应用 MVP**：Node.js + Express 后端，前端为原生 HTML/CSS/JS，数据用 JSON 文件存储（无数据库），界面为中文。
- 目录结构：`server.js`（后端 + 全部 API）、`public/`（前端 index.html / app.js / style.css）、`data/`（book.json / vote.json / user.json / settings.json）、以及 spec.md / task.md 等文档。
- 后端 API：图书列表、投票、注册/登录、管理员管理用户/图书/投票规则；两种投票规则 `single_total` 与 `single_per_book`。
- 指出几点注意事项：没有真正的会话认证（靠请求里的 username 识别身份，可伪造）、JSON 文件并发写有竞态风险（可能丢票）、vote.json 的 count 与 records 存在不一致、项目尚未初始化为 git 仓库。

**问题目的：** 先摸清项目的整体结构与现状，为后续改造（迁移数据库）打基础。
