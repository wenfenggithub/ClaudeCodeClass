## 当前实施进度（2026-07-03 更新）

> 本次构建产出：基于 **Next.js 14 + React 18 + TypeScript + Tailwind** 的 Web 原型。
>
> 启动方式：
> - `npm install && npm run dev` → http://localhost:3100
> - `npm run fetch-sounds` → 拉取 5 条基础环境录音（bradtraversy MIT 仓库）
> - `npm run import-sounds` → 扫 `~/Downloads` 把 Pixabay 下的 mp3 自动归位到 `public/sounds/`（按文件名关键词匹配 19 个目标槽位，回车一键确认）
> - `npm run test` → 跑 104 条单元测试
> - 直接打开 `hush.html` → 单文件离线预览（仅合成声音）
>

## 阶段四：进阶能力（可选）

### 目标
完成 spec.md 第 10.3 节的第三阶段功能：穿戴数据接入、端侧 AI 建议、云端同步、专家课程与更深的个性化能力，让产品具备长期竞争力。

### 任务清单

- [ ] 任务 1：智能手表 / 手环数据接入
  - 说明：对接 Apple Watch、小米手环、华为、OPPO 等主流设备的睡眠分期、心率、HRV 数据；处理多源数据冲突；用户可选择数据优先级。
  - 产出：穿戴用户可获得更精准的睡眠数据。

- [ ] 任务 2：端侧轻量睡眠监测（无穿戴）
  - 说明：基于手机加速度 + 麦克风做床上动作 / 鼾声粗略估计；全部计算端侧完成，原始数据不出端；明确标注为「估算」并附免责。
  - 产出：无穿戴用户可获得睡眠估算时长。

- [ ] 任务 3：端侧 AI 个性化建议
  - 说明：训练或集成一个轻量模型（用户分群、模式识别），用于挑选最相关的建议、生成更贴合的洞察文案；保留规则引擎作为兜底。
  - 产出：建议命中率与用户满意度有可量化提升。

- [ ] 任务 4：可选云端 LLM 洞察
  - 说明：默认关闭，用户主动开启；prompt 经医疗合规审查，强制不输出诊断 / 用药建议；上传内容仅为去 PII 的聚合特征。
  - 产出：高级用户可获得更自然语言化的洞察文案。

- [ ] 任务 5：端到端加密云同步
  - 说明：用户密钥不出端，服务端只存加密 blob 与最小元数据；支持多设备恢复；提供「关闭同步并删除云端数据」入口。
  - 产出：用户换机不丢失数据，且服务端无法读取明文。

- [ ] 任务 6：专家课程内容容器
  - 说明：搭建专家课程模块（视频 / 音频 / 图文混合）；课程必须经医疗合规审查通过后才能上架；不为单一医院 / 医生引流。
  - 产出：可承载专家课程的合规内容平台。

- [ ] 任务 7：节律与季节适配
