## 1. 项目概述

### 1.1 APP 名称（暂定）
- 中文名暂定：**「安眠岛」**（备选：晚安、睡眠岛、夜灯、Sleepwell、Hush）
- 英文名暂定：**Hush**
- Slogan：**"陪你慢慢睡着，一晚比一晚好。"**

### 1.2 产品定位
一款基于睡眠健康科学（睡眠卫生 + CBT-I 思路）的**非医疗类**睡眠改善辅助工具，帮助用户：
- 记录和理解自己的睡眠状况
- 通过行为干预、放松训练逐步改善睡眠
- 形成长期可坚持的健康作息习惯

**定位关键词**：温和、循序渐进、非焦虑、个性化、辅助工具（不是诊断工具）。

### 1.3 目标用户

| 用户类型 | 主要特征 | 占比预估 |
| --- | --- | --- |
| 入睡困难型 | 躺下 30 分钟以上才能睡着，常胡思乱想 | 35% |
| 易醒型 | 睡眠浅，半夜醒 1-3 次，难以再入睡 | 25% |
| 早醒型 | 比期望时间早醒 1-2 小时，无法再睡 | 10% |
| 作息紊乱型 | 周末熬夜补觉、跨时区、夜班、学生党、年轻白领 | 20% |
| 焦虑/压力型 | 工作压力、职场焦虑、考试焦虑导致失眠 | 10% |

**年龄段**：18–55 岁为主，核心用户 25–40 岁。
**性别**：男女均衡，女性略多（健康类 APP 通用规律）。
**设备**：以手机为主，**不依赖**可穿戴设备（首版）。

### 1.4 核心价值
1. **看得见的睡眠**：把抽象的"睡不好"变成可视化的数据。
2. **可执行的改善**：每天给出 1–3 条简单可做的建议，不堆砌信息。
3. **温柔陪伴感**：不打卡焦虑、不失败惩罚，强调"今晚比昨晚好一点"。
4. **隐私安全**：本地优先存储，不强制注册，不收集敏感生理数据。

### 1.5 非医疗免责声明（首屏 + 设置页常驻）
> **本应用是一款生活方式工具，不是医疗器械，不能用于诊断、治疗或预防任何疾病。**
> 应用中的建议基于公开的睡眠卫生原则与认知行为疗法（CBT-I）思路整理，不能替代医生、临床心理师或睡眠专科的诊疗。
> 若您出现持续 3 个月以上的失眠、严重日间功能损害、严重情绪低落、自伤念头、打鼾伴呼吸暂停等情况，请尽快前往医院睡眠科 / 精神心理科就诊。

## 3. 产品设计原则

| 原则 | 设计落点 |
| --- | --- |
| **低刺激** | 整体使用深色低饱和配色（夜间模式默认）、最小化弹窗、动画缓慢、字体较大 |
| **简单操作** | 每个核心场景操作 ≤ 3 步；睡眠日记 60 秒可填完 |
| **睡前友好** | 21 点后自动切到"夜间模式"：暗色 UI、暖色滤镜、动效降速、震动减弱 |
| **数据可视化** | 用图表代替数字、用情绪色而非红黄绿（避免"红色=失败"的焦虑暗示） |
| **循序渐进** | 计划默认 21 天为一个周期，每周只改 1 个变量 |
| **不制造焦虑** | 不展示"睡眠负债""睡眠债务""你已落后 X 天"等高压文案；用进步而非缺口表述 |
| **必要时建议就医** | 持续异常自动触发"温和但明确"的就医提示卡，不夸大、不弱化 |

## 4. 核心功能模块

### 4.1 睡眠档案（Profile）
**目标**：建立用户的基础画像，作为个性化建议的输入。

**字段**：
- 基本信息：昵称、性别、年龄段、身高（选填）、体重（选填）
- 作息偏好：常规上床时间、常规起床时间、目标睡眠时长
- 睡眠困扰类型（多选）：入睡难 / 易醒 / 早醒 / 多梦 / 打鼾 / 不知道
- 持续时长：< 1 个月 / 1–3 个月 / 3–6 个月 / 6 个月以上 / 多年
- 自评严重度：1–5 分 SLIDER
- 是否已就医、是否服药（选填，仅本地）
- 工作类型：固定时间 / 弹性 / 倒班 / 学生 / 自由职业
- 设备：是否使用智能手表/手环（用于决定数据来源）

**风险识别字段**（隐藏处理，触发就医提示）：
- 持续时长 ≥ 6 个月 + 自评 ≥ 4 → 触发"建议就诊"卡片
- 勾选"白天极度嗜睡 / 打鼾伴憋醒" → 触发"建议筛查睡眠呼吸暂停"

### 4.2 睡眠日记（Sleep Diary）
**目标**：CBT-I 中的核心工具，每天 60 秒完成。
**填写时机**：每天醒来后第一次打开 APP 时弹出（可关闭）；最长支持回填昨晚记录。

**早晨问卷字段**：
1. 昨晚上床时间（默认值 = 设置时间）
2. 大约多久睡着？（< 15 / 15–30 / 30–60 / > 60 分钟）
3. 夜间醒来次数（0 / 1 / 2 / 3+）
4. 醒来累计时长（< 5 / 5–20 / 20–60 / > 60 分钟）
5. 早晨起床时间
6. 主观睡眠质量（1–5 星）
7. 起床后状态（精神 / 一般 / 疲惫 / 极度疲惫）
8. 备注（可选，120 字以内）

**晚间问卷字段**（21 点提醒，可跳过；原型阶段以「回顾昨天」一并放入早晨问卷收集）：
1. 今天情绪（开心 / 平静 / 焦虑 / 低落 / 烦躁，多选）
2. 压力水平（1–5）
3. 咖啡因摄入（无 / 上午 / 下午 / 晚上）
4. 饮酒（无 / 少量 / 较多）
5. 是否运动（无 / 轻 / 中 / 强）
6. 最后一次屏幕使用时间（原型以「距上床 0–15 / 15–30 / 30–60 / >60 分钟」桶化收集，端侧反推 lastScreenTime）
7. 是否午睡及时长（桶：无 / ≤20 / 20–45 / >45 分钟）

**数据用途**：计算睡眠效率、生成报告、驱动建议引擎。

### 4.3 睡眠评分（Sleep Score）
**目标**：用一个 0–100 分让用户快速感知昨晚情况。

**评分维度（加权）**：
- 睡眠时长（25%）：与目标时长偏差越小分越高
- 睡眠效率（30%）：实际睡着时长 / 卧床时长
- 入睡潜伏期（15%）：< 20 分钟最佳
- 夜醒次数与时长（15%）
- 主观感受（10%）
- 作息规律性（5%）：与近 7 天平均上床/起床时间的偏差

**展示**：渐变色弧形进度条，配文案：
- 90–100：「美妙的一夜」
- 70–89：「不错」
- 50–69：「还可以」
- 30–49：「有点辛苦」
- 0–29：「今天对自己温柔一点」

> 文案绝不使用「失败 / 差 / 糟糕 / 红色警告」。

### 4.4 助眠音频（Sleep Sounds）
**目标**：提供放松、屏蔽环境噪声、形成睡眠条件反射。

**原型实现（V1.0）**：双轨道策略——优先用真人录制 mp3（放在 `public/sounds/{id}.mp3`），缺失时回退到 Web Audio 端侧合成。详见 `lib/audio-engine.ts`。

录制素材推荐来源：
- [Pixabay](https://pixabay.com/sound-effects/)（Pixabay Content License，可商用无需署名）
- [Mixkit](https://mixkit.co/free-sound-effects/)（Mixkit Free License）
- [Freesound.org](https://freesound.org)（搜索时过滤 CC0）
- 见 `public/sounds/README.md` 的下载清单和命名规范

**合成兜底**（不期待与录音音质对等，但保证 demo / 开发期可用）：
设计原则（柔和优先）：所有声音末端 LP ≤2.5kHz；无可听频段纯正弦波；事件类柔化；左右声道立体声差；10 秒噪音缓冲。

| 类型 | 实现 | 听觉特征 |
| --- | --- | --- |
| 雨声 | 双层粉噪 LP（1.2kHz L 声道 + 900Hz R 声道）+ 0.05Hz LFO ±8% 雨势微变 + 末端 LP 2kHz | 柔和持续唰唰，无嘀嗒声 |
| 海浪 | 棕噪 LP 500Hz 暗流 + 0.06Hz LFO 同时调主层 ±25% 和浪沫层 ±18% + 末端 LP 1.8kHz | ~16s 一次柔和呼吸式浪起落 |
| 风扇 | 棕噪 LP 90Hz sub-bass 隆鸣 + 粉噪 LP 700Hz 主体 + 3Hz LFO ±2.5% 叶片颤 + 末端 LP 1.5kHz | 纯噪音柔和呼呼，无电子音 |
| 森林夜虫 | 双层粉噪带通（1.5kHz/1.1kHz）左右声道 + 2.2kHz Q=18 白噪带通蝉鸣 + 12Hz 颤振 + 末端 LP 2.5kHz | 柔和远处虫鸣，无高频刺耳 |
| 篝火 | 棕噪 LP 250Hz 燃烧 + 粉噪 LP 800Hz 中频 + 0.4Hz 火苗摇曳 + 中低频带通柔和噼啪（极慢上升+衰减）+ 末端 LP 1.5kHz | 温暖低频，柔和噼啪 |
| 白 / 粉 / 棕噪音 | Paul Kellet 算法 + 末端 LP（白 6k / 粉 5k / 棕 2k） | 纯色噪音 |

**睡前故事**：6 篇原创短句体故事 + **双引擎朗读**（登录后可用）
- 6 篇原创短文（《湖边的灯》《一只不再赛跑的乌龟》《风的一晚》《星星和云》《一条很慢的河》《雪夜》），按现有 TTS 估算规则校准到 UI 显示时长：《湖边的灯》《风的一晚》约 4 分钟，其余约 5 分钟
- 优先 **阿里云百炼 DashScope CosyVoice v3**（默认）：
  - HTTP endpoint：`https://dashscope.aliyuncs.com/api/v1/services/audio/tts/SpeechSynthesizer`
  - 通过 Next.js API 路由 `/api/tts` 代理（CORS + 凭证保护），GET 探测与 POST 合成均需登录
  - 凭证：`.env.local` 配置 `ALIYUN_DASHSCOPE_API_KEY`
  - 12 个 v3 音色（龙安洋/龙安柔/龙安灵/龙安雅/龙安亲/龙妙/龙媛/龙悦/龙婉君/龙书/龙硕 + default 映射）
  - 非流式合成：POST → 取音频 URL → 下载 mp3 返回
  - IndexedDB 缓存：key 包含 engine + voice + speed + story + paragraph + 文本指纹
- 可选备用 **Microsoft Edge TTS**：
  - 需安装 Python `edge-tts` 库
  - 8 个中文神经语音（晓晓/晓辰/晓伊/云希/云健/云扬/晓臻/曉佳）
  - 浏览器 Web Speech API 备用（rate=1.0 / pitch=1.0）
- **无自动 fallback**：引擎失败直接显示错误，不自动切换
- 优先选中文女声（macOS Tingting/Mei-Jia 等）
- 登录后可与环境声叠加播放（雨声 + 故事）；匿名入口置灰提示「登录后可用」
- 无中文语音的设备降级为"暂不可用"提示

**未来内容（V1.5+，等采购方案）**：
- ASMR（耳语、翻书、键盘）
- 钢琴 / 环境音乐 / Lo-Fi
- 真人录制故事（10/20/30 分钟版本）— 替换当前 TTS 版本
- 引导式入睡（10–25 分钟语音引导）

**功能**（V1.0 已实现 ✅）：
- ✅ 定时关闭（5 / 10 / 15 / 30 / 45 分钟，1.5s 淡出；MiniPlayer 按钮内联倒计时显示）
- ✅ 多轨混音（最多 3 轨叠加，独立音量；超过自动移除最早一轨）
- ✅ 收藏 / 最近播放（持久化到 localStorage）
- ✅ "今晚的推荐"：基于昨晚日记的情绪 / 夜醒 / 主观质量生成
- ⏸ 离线下载（采购素材后再做，会员功能）
- ⏸ 检测到睡着时停止（V2，需手机加速度计）

### 4.5 呼吸训练（Breathing）
**目标**：通过节律呼吸激活副交感神经，加速入睡。

**访问边界**：登录后可用；训练页入口保持可见，未登录时置灰提示。

**训练类型**（4 种均已落地）：
- 4-7-8 呼吸法（睡前主推，默认 5 分钟）
- 箱式呼吸（4-4-4-4，默认 5 分钟）
- 腹式呼吸（4-6 节律，默认 6 分钟）
- 共振呼吸（5.5 次/分钟 = 吸 5.5s / 呼 5.5s，默认 7 分钟）

**交互**：
- 屏幕中央一个缓慢呼吸的圆球（吸气放大、屏息保持、呼气缩小、屏息收缩四相）
- 节律配置以毫秒为单位，支持非整数秒（共振呼吸 5.5s）
- 配可选低频引导音 + 触觉震动（v2 接入）
- 默认 5–7 分钟，可自定义 1–20 分钟（自定义留 v2）
- 完成后给一句"今晚也辛苦了"式文案，不打分

### 4.6 冥想放松（Meditation & Relaxation）✅ V1.0
**目标**：舒缓焦虑、卸载白天压力。

**内容**（6 篇原创引导脚本已实现，`lib/meditations.ts`）：
- ✅ 渐进式肌肉放松（PMR）：逐组收紧→放开（`med-pmr`，约 12 分钟）
- ✅ 身体扫描（Body Scan）：从头顶扫到脚趾（`med-bodyscan`，约 10 分钟）
- ✅ 正念呼吸：5 / 10 / 15 分钟三档（`med-mind-5/10/15`）；5 分钟为「短暂停顿 / 呼气末端留白」，10 分钟为睡前安顿，15 分钟为深度放松，并用显式静默停顿接近真实 5/10/15 分钟练习时长
- ✅ 担忧卸载引导（Worry Dump）：把烦心事"装进盒子"安放（`med-worry`，约 8 分钟）；与 §4.8 担忧时间的写入式工具互补
- 助眠故事（与音频模块联动，见 §4.4）

**访问边界**：登录后可用；训练页入口保持可见，未登录时置灰提示。

**技术实现**：
- 引导词为原创短句口语体，按段落组织（段间自带停顿），与睡前故事同构
- **复用现有 TTS 引擎**（`lib/tts-engine.ts` 的 `resolveSpoken` 同时解析故事与冥想 id；冥想 id 统一 `med-` 前缀避免碰撞）：阿里云 CosyVoice v3（默认）/ Edge TTS / 浏览器原生，IndexedDB 缓存复用；冥想整体登录后可用
- 正念脚本支持 `[[pause:N]]` 显式静默停顿；云端 TTS 开启时逐段合成非标记文本，并由播放器插入静默，避免云端合成读出控制标记
- MP3 缓存 key 含 engine / voice / speed / content id / 文本指纹；首次播放写入 IndexedDB，二次播放同内容复用缓存，内容更新自动生成新 key
- 播放页 `app/practice/meditation`：呼吸式光球 + 段落进度 + 暂停/继续；完成后记 `PracticeSession`（type=meditation/pmr），文案"今晚也辛苦了"不打分
- 入口：训练页「冥想 · 放松」分区；未登录时列表保留但置灰提示「登录后可用」，登录后进入播放页

### 4.7 睡眠习惯计划（Sleep Plan）
**目标**：把"想睡好"拆解成 21 天微习惯计划。

**生成逻辑**：
- 根据睡眠档案 + 前 7 天日记，AI/规则引擎生成个性化计划
- 每周 1 个"主任务" + 2–3 个"微习惯"

**示例 21 天计划**：
- 第 1 周：固定起床时间（每天同一时间起床，包括周末）
- 第 2 周：建立"床=只睡觉"刺激控制（不在床上刷手机）
- 第 3 周：睡眠限制（只在感到困倦时上床，提升睡眠效率）

**呈现**：日历视图 + 每日任务卡片（≤ 2 分钟可完成）。

  3. 点击底部播放条 → 进入全屏播放页（极简，仅显示动画 + 定时器）

### 5.4 训练页（Practice）
- **页面目标**：呼吸 + 冥想 + 放松练习的统一入口。
- **主要功能**：训练分类、收藏、最近练过、引导式入睡专区
- **关键组件**：练习卡片（封面图、时长、是否会员）、播放页、完成页（无评分，仅文案）
- **用户操作流程**：
  1. 选练习 → 进入准备页（"找一个舒服的姿势"）
  2. 倒计时 3 秒开始 → 全屏引导
  3. 完成 → "今晚也辛苦了" → 推荐下一步（睡眠日记 / 入睡音频）

### 5.5 睡眠计划页（Plan）
- **页面目标**：让用户看到自己正在执行的改善计划。
- **主要功能**：当前计划日历、今日任务、计划详情、暂停/重启计划、切换计划

  1. 进入 → 看到今天的 1–3 个微任务
  2. 完成任务 → 打勾（不显示连击/惩罚）
  3. 点击日历 → 查看任意一天的计划与完成情况

### 5.6 睡眠报告页（Reports）
- **页面目标**：让用户看懂自己的睡眠模式。

- **用户操作流程**：
  1. 进入 → 默认显示本周
  2. 切换"日 / 周 / 月"
  3. 点击图表节点查看当天详情
  4. 点击底部"分享/导出" → 生成图片或 PDF

### 5.7 个人中心（Profile）
- **页面目标**：管理档案、会员、设备、内容收藏。

## 6. 用户流程

### 6.1 首次使用流程（Onboarding）
1. 启动 → 简短动画（≤ 3 秒）
2. 4 张引导卡片（不超过 4 张）
3. **医疗免责声明** + 必须勾选"我已知晓"
4. 首页 / 引导页顶部提供账户卡片；"先体验"仍为默认推荐，本地匿名 ID 可使用基础功能
5. 睡眠档案问卷（10 题，约 90 秒）
6. 风险初筛（持续时长 + 严重度），命中即推送就医提示卡
7. **推荐首个 21 天计划** ✅ V1.0：独立插页展示三周主题概览 + [加入] / [先看看]，加入即创建默认 plan
8. 进入首页 → 自动播放一次"使用引导小贴士"

### 6.2 每晚睡前流程
1. 21:00 提醒："是否进行睡前放松？"（可关闭）
2. 用户点击 → 推荐 1 个呼吸练习 + 1 个冥想 + 助眠音频
3. 完成训练 → 提示开启"屏幕宵禁"
4. 进入"夜灯模式"（极暗 UI，仅显示音频播放）
5. 设置起床闹钟（如未设置）

### 6.3 每天醒后流程
1. 闹钟响起 → 渐亮屏幕 + 渐强自然声
2. 关闭闹钟后 → 自动弹出睡眠日记（早晨问卷）
3. 提交后 → 显示昨晚评分 + 一句洞察
4. 推荐今日的 1–3 个微任务
5. 引导回首页

### 6.4 每周复盘流程
1. 每周日（或用户自定义日）晚上 20:00 推送周报
2. 用户进入周报页 → 阅读趋势 + 行为相关性洞察
3. 系统询问："下周你想优先改进哪一项？"（提供 2–3 个选项）
4. 用户选择 → 自动调整下周计划

### 6.5 长期改善流程（21 天 + 周期循环）
- D0：完成档案 + 选择计划
- D1–D7：第 1 周主题（如固定起床时间）
- D7：第 1 周复盘 + 自动调整
- D8–D14：第 2 周主题
- D15–D21：第 3 周主题
- D21：21 天总结报告 + 是否进入下一个 21 天周期
- 每月：月报 + 是否升级会员获取深度分析

## 8. 个性化建议逻辑

### 8.1 建议引擎架构
采用 **规则引擎（v1）+ 规则 + 轻量 ML（v2+）** 的混合方案：
- 规则引擎可解释、可审核，符合健康类合规要求
- 规则触发后挑选 1–3 条最高优先级的洞察展示

### 8.2 核心规则（节选）

| 规则 ID | 触发条件 | 输出建议（示例文案） | 优先级 |
| --- | --- | --- | --- |
| LATE_BEDTIME_3 | 近 3 天上床时间 > 目标 + 60 分钟 | "最近上床都比平时晚了一个多小时，明晚可以试试 23:00 前关灯" | 中 |
| SHORT_SLEEP_3 | 近 3 天睡眠时长 < 6h | "你已经连续 3 天睡得有点少，今天可以提前 30 分钟准备睡觉" | 高 |
| LONG_LATENCY | 近 3 天入睡 > 30 分钟 | "你的入睡时间偏长，要不要试试 4-7-8 呼吸？通常 5 分钟就有效果" | 高 |
| FREQUENT_AWAKENING | 近 7 天平均夜醒 ≥ 2 次 | "最近夜里醒得多，留意一下傍晚后是否摄入了咖啡因或酒精" | 中 |
| HIGH_STRESS_5 | 近 5 天压力 ≥ 4 | "最近压力指数偏高，睡前 10 分钟的'担忧时间'可能会帮到你" | 高 |
| LATE_SCREEN | 近 5 天最后屏幕时间 < 上床前 30 分钟 | "在睡前 30 分钟收起手机，可能会让入睡快很多" | 中 |
| LATE_CAFFEINE | 近 7 天有 ≥ 3 天下午 3 点后摄入咖啡因 | "下午 3 点后的咖啡因，可能正在偷走你的深睡眠" | 中 |
| ALCOHOL_HIGH | 近 7 天有 ≥ 3 天饮酒"较多" | "酒精虽然让你更快睡着，但会显著降低后半夜的睡眠质量" | 中 |
| WEEKEND_SHIFT | 周末上床比工作日晚 > 90 分钟 | "周末熬夜会打乱你的生物钟，试试把差距缩小到 1 小时内" | 低 |
| LOW_EFFICIENCY_14 | 近 14 天效率 < 65% | 提示考虑医疗咨询（warning 级） | 高 |
| MOOD_LOW_30 | 近 30 天"低落"出现 ≥ 20 次 | 温和提示心理科 | 高 |

### 8.3 反焦虑规则
- 单日只展示 ≤ 3 条洞察
- 同一规则 7 天内不重复展示
- 文案模板必须以"鼓励 + 行动"结尾，避免"你做错了"
- "你已连续 X 天没…"类型的文案禁用，改为"再坚持一次就有 N 天了"

## 9. 医疗风险与边界

### 9.1 明确的产品边界
| 我们做 | 我们不做 |
| --- | --- |
| 提供生活方式改善建议 | 提供任何疾病诊断 |
| 引用公开的睡眠卫生 / CBT-I 思路 | 提供处方药、剂量建议、停药指导 |
| 鼓励用户记录、复盘、放松 | 承诺"治好失眠""治愈抑郁" |
| 在风险信号下推荐就医 | 替代医生、心理师 |

### 9.2 强制就医提示场景

| 场景 | 提示等级 | 具体动作 |
| --- | --- | --- |
| 持续 3 个月以上失眠 + 严重日间损害 | warning | 顶部常驻"建议就诊"卡片 + 资源链接 |
| 打鼾 + 白天严重嗜睡 + 憋醒 | warning | 提示筛查睡眠呼吸暂停（科普 + 就诊建议） |
| 长期情绪低落（≥ 2 周抑郁倾向） | warning | 推送精神心理科信息 + PHQ-9 自评（可选） |
| 备注/聊天中出现自伤/自杀关键词 | urgent | 立即弹出心理援助热线，提供"立即拨打" |
| 怀孕、青少年（年龄段 < 18）使用 | mild | 限制部分功能（如睡眠限制疗法）+ 提示与医生沟通 |

### 9.3 文案合规
- 所有文案需经过"医疗合规审查"，禁止使用："治愈、根治、有效率 X%、临床证明、医院推荐"等绝对化表述
- 如引用研究文献，需在脚注列出来源

### 9.4 法规与认证
- 中国大陆：避免被认定为二类医疗器械（不做诊断、不输出治疗方案）
- 数据合规：《个人信息保护法》《数据安全法》、健康数据特殊保护
- 海外（如 Apple App Store 健康类条款）：需符合 Health & Fitness 类目而非 Medical 类目

- 不卖助眠保健品（褪黑素、营养品）
- 不接入第三方广告 SDK（健康类用户对广告敏感度高）
- 不做"焦虑营销"

## 12. 技术实现思路

### 12.1 前端
- **框架**：跨平台优先 → **React Native** 或 **Flutter**（推荐 Flutter，对动画/夜间模式更友好）
- **状态管理**：Riverpod（Flutter）或 Zustand（RN）
- **本地存储**：SQLite（Drift / WatermelonDB）+ SQLCipher 加密
- **音频播放**：just_audio / react-native-track-player，支持后台 / 锁屏 / 多轨混音
- **动画**：Lottie + Rive（呼吸球动画，低性能消耗）
- **可访问性**：支持系统字号、对比度模式、VoiceOver / TalkBack

### 12.2 后端
- **架构**：服务端尽量轻量（账号、订阅、内容分发、同步）
- **服务**：FastAPI / Node.js（NestJS）+ Postgres + Redis + 对象存储（音频）
- **认证**：手机号 / Apple ID / 微信 / 匿名升级
- **订阅**：StoreKit 2（iOS）+ Google Play Billing + 服务端校验
- **CDN**：音频走 CDN，分国内 / 海外两套
- **隐私设计**：
  - 默认匿名，不强制登录；匿名可用基础功能，故事 / 放松训练 / 三方 TTS / 管理员功能需登录
  - 同步可关闭
  - 服务端不存储日记原文（如开启同步，只存哈希 + 端到端加密 blob）

#### 12.2.1 原型阶段账户实现（✅ V1.0）
> Web 原型用最轻量的方式验证账户/管理员产品逻辑，接口与上述生产设计对齐，便于后续平移。

- **存储**：JSON 文件 `.data/users.json`（gitignore），模块级写互斥锁；只存 `email / passwordHash / role / disabled / localUserId / 时间戳`，**不含任何日记或健康明文**。生产迁 Postgres。
- **密码**：Node 内置 `crypto.scrypt`（`scrypt$N$r$p$salt$hash`），`timingSafeEqual` 校验。
- **会话**：无状态签名 httpOnly cookie（HMAC-SHA256，`AUTH_SECRET`），`sameSite=lax`，30 天；登出即清 cookie。
- **密码找回**：邮箱重置链接（令牌仅存 SHA-256 哈希、30 分钟过期、单次使用）；配置 `SMTP_*` 后真实发送邮件（465 TLS / 587 STARTTLS），未配 SMTP 时仅打印到服务端控制台；`/api/auth/forgot` 不向浏览器返回链接，恒返回 200 防用户枚举。
- **匿名升级**：注册/登录时把本地匿名 `profile.userId` 作为 `localUserId` 关联到账户，本地数据仍是唯一真源。
- **账户入口**：登录 / 注册 / 找回 / 重置统一整合到首页账户卡片；旧 `/login`、`/register`、`/forgot`、`/reset` 仅作为兼容入口重定向到首页对应模式；登录成功后按 `next` 回到用户原本想访问的功能页。
- **匿名 / 登录边界**：匿名保留引导、档案、日记、评分、报告、计划、助眠声音四类各 1 个试听、本地导出/清空；其余声音、睡前故事、放松训练（呼吸、冥想、担忧时间、认知重构）、阿里云 TTS、Edge TTS 与管理员控制台登录后可用。
- **角色与管理员**：首个注册用户或 `SEED_ADMIN_EMAIL` 命中 → `admin`；管理 API 服务端 `requireAdmin` + 前端路由守卫双重校验；防锁死（最后一个管理员不可停用/删除）。
- **API**：`/api/auth/{register,login,logout,me,forgot,reset}`、`/api/admin/{users,users/[id],users/[id]/reset-password,overview,content}`，均 `runtime=nodejs`。
- **限制**：以上需 server 模式；`STATIC_EXPORT` 静态导出与 `hush.html` 单文件版无 API 路由，账户、管理员、故事与放松训练不可用。

### 12.3 数据库
- 端：SQLite + SQLCipher
- 云：Postgres（强类型 + 报表友好）
- 缓存：Redis（订阅校验、限流）
- 大数据（可选）：仅聚合的非 PII 数据，用于产品分析

### 12.4 音频播放
- 支持后台播放、锁屏控制、AirPlay / 蓝牙
- 多轨叠加（最多 3 轨）
- 定时关闭：本地 Timer + 系统级后台音频会话保活
- 离线缓存：分段下载 + 端侧加密（防止被外部播放器扒取）

### 12.5 通知与提醒
- 本地通知（不依赖服务端推送，避免隐私风险）
- 关键提醒：上床、起床、屏幕宵禁、睡眠日记、计划任务
- "夜间静音模式"：22:00–7:00 不发非关键推送
- 服务端推送（远程）仅用于：会员到期、系统维护，频率严格限制

### 12.6 数据可视化
- 图表库：fl_chart（Flutter）/ Victory Native（RN）
- 配色：低饱和、夜间友好、不用红色表达"差"
- 关键图：睡眠评分折线、上床/起床散点（节律图）、行为热力图

### 12.7 AI 建议模块
- **V1**：纯规则引擎（YAML 配置规则，便于运营/医疗顾问审核）
- **V2**：规则 + 端侧轻量模型（用户分群 / 模式识别）
- **V3**：可选云端 LLM 生成"洞察文案"，但需：
  - 医疗合规审查 prompt
  - 强制免责声明
  - 不输出诊断 / 用药建议
  - 默认关闭，用户主动开启

### 12.8 隐私与安全
- 所有日记、备注、风险标记 → **本地加密**
- 关键词匹配 → **端侧完成**，不上传内容
- 同步采用 **端到端加密**（用户密钥不出端）
- 删除账号 → 服务端 7 天内彻底删除
- 隐私合规：GDPR / CCPA / 中国《个人信息保护法》
- 第三方 SDK 白名单：仅崩溃监控（不含 PII）+ 订阅 SDK；**默认不集成行为埋点 SDK**，自建匿名指标
